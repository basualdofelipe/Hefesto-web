  <action>
    **SupplyTable.tsx** (main client component):
    - Props: supplies (with currentPrice/lastPriceUpdate), supplyTypes, suppliers
    - State: searchQuery (string), selectedSupplierId (string | null), showInactive (boolean, default false)
    - Search: filter supplies by name (case-insensitive includes). Debounce not required for initial implementation (discretion).
    - Supplier filter: dropdown select from suppliers list. "Todos" option clears filter.
    - "Mostrar inactivos" toggle (Switch component) — off by default, only shows active supplies.
    - Group filtered supplies by type.name. Render one SupplyTypeGroup per group.
    - "Nuevo insumo" button opens SupplyFormDialog in create mode. Visible only to admin (useIsAdmin hook).
    - After any mutation (create, edit, toggle, add price), call `router.refresh()` to revalidate server data.

    **SupplyTypeGroup.tsx** (collapsible section per type):
    - Props: typeName, supplies array, count
    - Uses Shadcn Collapsible component. Each group manages its own open/closed state independently (per Pitfall 5).
    - Header: type name + count badge (e.g., "Cuero (3 insumos)"). ChevronDown icon rotates on open.
    - Default state: all groups open (discretion).
    - Inside: Table with columns: Nombre, Proveedor, Precio Actual, Estado.
    - Precio Actual: format with `toLocaleString('es-AR')` and unit suffix (e.g., "$15.200/m2"). Show "Sin precio" if no current price.
    - Estado: Badge "Activo" (green) or "Inactivo" (red). Inactive rows have `opacity-50` class.
    - Click on a row toggles expand (not the entire row — use a chevron button or the row itself).

    **SupplyExpandedRow.tsx** (inline expanded content):
    - Shows below the clicked row inside the table.
    - Content:
      - Notas: text from supply.notes (or "Sin notas" placeholder)
      - Proveedor: clickable link navigating to `/proveedores/${supply.supplier.id}/editar`
      - Precio actual: amount with unit (e.g., "$15.200/m2") and "Ultima actualizacion: {date}"
      - Action buttons (admin only):
        - [Editar]: opens SupplyFormDialog in edit mode
        - [+ Nuevo precio]: toggles AddPriceInline visibility
        - [Ver historial]: opens PriceHistoryDialog
        - [Toggle switch]: calls PATCH toggle-status endpoint, shows toast
    - When AddPriceInline is visible, it appears below the actions.

    **SupplyFormDialog.tsx** (create/edit modal):
    - Uses Shadcn Dialog + react-hook-form + zodResolver
    - Zod schema:
      - name: z.string().min(1, 'El nombre es obligatorio').max(255)
      - typeId: z.string().uuid('Selecciona un tipo')
      - supplierId: z.string().uuid('Selecciona un proveedor') — only in create mode
      - unitType: z.enum(['m2', 'unidad', 'metro', 'kg'])
      - notes: z.string().optional().or(z.literal(''))
      - initialPrice: z.number().min(0).optional() — only in create mode
    - Create mode: POST /supplies with all fields. supplierId select enabled.
    - Edit mode: PUT /supplies/:id with name, typeId, unitType, notes. Supplier shown as read-only text (NOT editable — locked decision). No initialPrice field.
    - On success: toast.success, close dialog, router.refresh().
    - On error: toast.error with message from API response.

    **PriceHistoryDialog.tsx** (price history modal):
    - Opens via "Ver historial" button in expanded row.
    - Fetches GET /supplies/:id/prices on open (using apiClientFetch).
    - Renders a simple list: each row shows date (formatted dd/mm/yyyy) and price with unit.
    - Sorted newest to oldest (API returns this order).
    - No visual indicators (no arrows, colors, percentages — locked decision: deferred).
    - Loading state with Skeleton while fetching.
    - Empty state if no prices.

    **AddPriceInline.tsx** (inline price input):
    - Appears below the action buttons when "+ Nuevo precio" is clicked.
    - Single number input + confirm (Check icon) + cancel (X icon) buttons.
    - On confirm: POST /supplies/:id/prices with { price: number }.
    - On success: toast.success("Precio actualizado"), hide input, router.refresh().
    - On error: toast.error.
    - Input uses type="number" with step="0.01" and min="0".

    **TypeScript types** (define at top of SupplyTable.tsx or in a shared types file):
    ```typescript
    interface SupplyType { id: string; name: string; }
    interface Supplier { id: string; name: string; }
    interface Supply {
      id: string;
      name: string;
      type: SupplyType;
      supplier: Supplier;
      unitType: 'm2' | 'unidad' | 'metro' | 'kg';
      notes: string | null;
      isActive: boolean;
      currentPrice: number | null;
      lastPriceUpdate: string | null;
      createdAt: string;
    }
    interface PriceRecord {
      id: string;
      price: string; // decimal from backend
      createdAt: string;
    }
    ```

    **Unit display mapping:**
    - m2 -> "m2" (or "m\u00B2")
    - unidad -> "un."
    - metro -> "m"
    - kg -> "kg"
