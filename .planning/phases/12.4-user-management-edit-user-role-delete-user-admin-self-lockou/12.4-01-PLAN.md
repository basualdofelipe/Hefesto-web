## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → `/users/:id` PATCH | Untrusted input (JSON body with name/roleId/isActive) crosses HTTP boundary into NestJS validation pipeline |
| Client → `/users/:id` DELETE | Untrusted UUID param + caller identity from JWT (forged JWT risk handled by JwtAuthGuard upstream) |
| Service ← Controller | Trusted boundary: controller passes only validated dto + JWT-derived callerId |
