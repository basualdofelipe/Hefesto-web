## What Was Fixed

**BLOCKER 1 (UAT Tests 2 + 8) — FOR UPDATE + COUNT illegal in PostgreSQL:**

`countOtherActiveAdmins` in `users.service.ts` combined `.setLock('pessimistic_write')` (which emits `FOR UPDATE`) with `.getCount()` (which emits `COUNT(DISTINCT)`). PostgreSQL prohibits `FOR UPDATE` with aggregate functions — this caused a `500 QueryFailedError` + ROLLBACK on every admin demote/deactivate/delete operation. The fix is a single-line removal: the `SERIALIZABLE` transaction isolation on `update()` and `remove()` already serializes concurrent requests without needing a row-lock on the COUNT query.

**BLOCKER 2 (UAT Test 6) — TypeORM property name vs. column name in `.set()`:**

`transferOwnership` in `scenarios.service.ts` used `updated_at: () => 'NOW()'` in the QueryBuilder `.set()` object. TypeORM's `.set()` resolves keys by **entity property names**, not DB column names. The `@UpdateDateColumn` property on `BaseEntity` is named `updatedAt` (mapped to column `updated_at`). Passing `updated_at` caused `EntityPropertyNotFoundError: Property 'updated_at' was not found in 'Scenario'` → 500 + ROLLBACK on every user delete that had scenarios. Fixed by changing the key to `updatedAt`.

## E2e Test Design Notes

The e2e spec at `test/users-admin-guard.e2e-spec.ts`:
- Imports `AppModule` (same as `app.e2e-spec.ts`) — runs real migrations via `migrationsRun: true`, so the seed admin `basualdofelipe@gmail.com` is always present.
- Calls services directly (not HTTP) to avoid JWT/auth harness complexity.
- Uses a `TEST_PREFIX = 'e2e-admin-guard-'` for all test-created rows — `afterEach` deletes by prefix, never touching the seed admin except to restore `is_active = true`.
- Test 2 temporarily deactivates the seed admin to establish the "exactly 1 active admin" precondition, then restores it in `afterEach`.
- Requires `docker compose up -d postgres-test` (postgres-test on port 5433).
