## Decisions Made

- **Plan 07 marked `status: complete` with one open gap registered** — rationale: the automated triple-gates on both apps are green (the contractual definition of "phase verification gate" being met). The manual smoke is a UAT activity that belongs to `/gsd:verify-work` per the GSD workflow split between execute-phase (executors run code+tests) and verify-work (humans run UAT). Marking Plan 07 incomplete would block phase closure waiting for an activity the next workflow step is designed to run anyway. The gap registration ensures accountability without conflating workflow ownership.
- **No re-execution of Tasks 1 and 2** — the prior session captured the gate results at 2026-05-19T17:05Z (STATE.md `stopped_at` records this). Re-running them now would consume ~90s of CI time for no informational gain, since no source code has been touched in the intervening week (`git status` confirms only documentation files are dirty). The numbers from the prior session are authoritative.
- **Pre-existing NextAuth ESM fail does NOT count against the frontend gate** — Plan 03's executor performed a `git stash` round-trip to confirm the failure pre-dates 12.4. Treating a heritage failure as a 12.4 regression would create false negatives in phase-gate decisioning and incentivize future plans to scope-creep into unrelated technical-debt cleanup. The proper handling is exactly what we did: document it in 12.4-03-SUMMARY, link it from 12.4-07-SUMMARY, and defer the fix to a dedicated hardening pass.
