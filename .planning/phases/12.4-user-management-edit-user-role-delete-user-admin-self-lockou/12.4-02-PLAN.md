## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller (UsersService) → ScenariosService.transferOwnership | Trusted boundary inside backend; caller has already passed auth + permission checks |
| ScenariosService → DB via QueryRunner | Trusted boundary; QueryRunner is a typed TypeORM construct that parametrizes SQL |
| User-controlled `suffix` value (originally `victim.name`) → SQL literal | Critical: suffix value originates from DB (victim user.name field) but is interpolated into SQL — MUST use parameter binding |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12.4-06 | Tampering / SQL Injection | `transferOwnership` bulk UPDATE | mitigate | Use `setParameter('suffix', suffix)` instead of string concatenation. Arrow function `() => "LEFT(COALESCE(name, '') || :suffix, 200)"` keeps the placeholder unbound until TypeORM's parameter binder fills it. Even if `victim.name` contained SQL fragments like `'; DROP TABLE users;--`, it would be inserted as a literal string value. Verified by RESEARCH Finding 2 + TypeORM issue #1580. |
| T-12.4-11 | Tampering | `where('user_id = :victimId', { victimId })` | mitigate | victimId is also parametrized inline. ParseUUIDPipe in the caller controller (Plan 01) already validates UUID shape before this method receives it. |
| T-12.4-12 | Denial of Service | bulk UPDATE on large scenarios table | accept | Phase scale is low (single-digit users, ≤100s scenarios per user). UPDATE with FK index on `user_id` is cheap. No mitigation needed. |
| T-12.4-13 | Information Disclosure | error propagation when `execute()` fails | accept | Errors bubble to caller's catch which rollbacks. NestJS framework converts to 500 (or 400 if BadRequestException). The TypeORM error message could leak schema info — but only an admin caller (gated by Plan 01's `@RequirePermission('can_manage_users')`) can trigger this path. |
| T-12.4-SC | Tampering | npm/pip/cargo installs | n/a | No package installs in this plan — uses existing typeorm and @nestjs/testing only. |
</threat_model>
