## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12.4-04 | Elevation of Privilege | self-delete UI bypass | mitigate (UI layer) | Borrar button on own row is HIDDEN by UsersClient (Plan 06) — dialog never opens for own user. Backend guard (Plan 05) is the authoritative check; UI hide is UX courtesy. |
| T-12.4-18 | Tampering | DELETE method via mass-trigger | accept | AlertDialog modal-blocking nature prevents accidental confirms. `e.preventDefault()` on action onClick prevents Radix auto-close-then-action sequence. |
| T-12.4-19 | Information Disclosure | error toast | accept | Backend Spanish messages are informational. Generic 500 fallback is "No se pudo eliminar el usuario". |
| T-12.4-20 | XSS | rendering user.email and user.name in JSX | mitigate | React escapes text children by default. `<strong>{user.email}</strong>` and `{user.name ?? 'usuario borrado'}` are text nodes — safe. No `dangerouslySetInnerHTML`. |
| T-12.4-21 | Spoofing | clicking Borrar with stale user data | accept | Parent (UsersClient) controls `user` prop. If the underlying user was deleted in another tab, the next DELETE will return 404 — handled gracefully via toast.error. |
| T-12.4-SC | Tampering | npm install for testing | n/a | No new packages — uses existing @testing-library and jest infrastructure. |
