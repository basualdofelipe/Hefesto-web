# Phase 12.4 Plan 01: User management HTTP surface (DTO + Controller) Summary

**One-liner:** Sets up the HTTP contract for the user-management refactor — adds `UpdateUserDto` and exposes new `PATCH /users/:id` + `DELETE /users/:id` endpoints, while deleting the legacy `toggle-status` route.

## What Was Built

### DTO — `nemea-back/src/users/dto/update-user.dto.ts` (NEW)

`UpdateUserDto` with three optional fields, all decorated with `class-validator` + `@ApiProperty`:

| Field      | Type      | Validator      | Required |
|------------|-----------|----------------|----------|
| `name`     | `string`  | `@IsString()`  | optional |
| `roleId`   | `string`  | `@IsUUID()`    | optional |
| `isActive` | `boolean` | `@IsBoolean()` | optional |

Each field carries `@IsOptional()` and has a Spanish `@ApiProperty({ required: false, description: ... })` Swagger annotation. Field order (`name → roleId → isActive`) matches D-17 UI form order.

**Excluded by design:**
- `email` — OAuth identity / whitelist key (SPEC Out of Scope).
- `pictureUrl`, `googleId` — sourced from Google, overwritten on next signin.
- **No `PartialType(CreateUserDto)`** — heredaría `email` + `@IsNotEmpty()` que rompen el contrato de "todos optional".

### Controller — `nemea-back/src/users/users.controller.ts` (MODIFIED)

**Removed:**
- Entire `@Patch(':id/toggle-status')` block (D-12).
- Inline self-check `if (id === currentUser.id) throw BadRequestException(...)` (logic moved to service per D-04/D-05).
- Unused imports: `BadRequestException`, `NotFoundException`, `JwtUser`.

**Added:**
- `@Patch(':id')` endpoint:
  - `@RequirePermission('can_manage_users')`
  - `@ApiOperation + @ApiResponse(200/400/404/403)`
  - Signature: `async update(id, dto: UpdateUserDto, callerId: string): Promise<User>`
  - Body: pure delegate `return this.usersService.update(id, dto, callerId)`
- `@Delete(':id')` endpoint:
  - `@HttpCode(204)` — REQ-12.4-3 contract
  - `@RequirePermission('can_manage_users')`
  - `@ApiOperation + @ApiResponse(204/400/404/403)`
  - Signature: `async remove(id, callerId: string): Promise<void>`
  - Body: pure delegate `return this.usersService.remove(id, callerId)`

### Final controller endpoint roster (4 endpoints)

| Method | Path             | Permission             | Notes                                         |
|--------|------------------|------------------------|-----------------------------------------------|
| GET    | `/users`         | `can_manage_users`     | unchanged                                     |
| POST   | `/users`         | `can_manage_users`     | unchanged                                     |
| PATCH  | `/users/:id`     | `can_manage_users`     | **NEW** — replaces `toggle-status`            |

## Decisions Made

- **No `PartialType(CreateUserDto)`** for `UpdateUserDto` — would carry `email` and `@IsNotEmpty()` on `roleId`, breaking the all-optional contract.
- **`@HttpCode(204)` explicit on DELETE** — NestJS defaults to 200 with no body; without this decorator REQ-12.4-3 (204 No Content) would fail in the integration test surface that Plan 06 builds.
- **`@CurrentUser('id') callerId: string`** instead of `@CurrentUser() user: JwtUser` — matches D-04 (service receives only `callerId` string, not the full JWT payload).
- **Unused imports purged at task close** — `BadRequestException`, `NotFoundException`, `JwtUser` removed once the inline self-check moved out of the controller; this prevents the next ESLint run from flagging them.
