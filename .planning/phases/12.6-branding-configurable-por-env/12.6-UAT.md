## Tests

### 1. Cold Start Smoke Test
expected: Con una DB fresca, levantar back (`npm run start:dev`) + front (`npm run dev`) desde cero. Las migraciones corren y siembran admin (`admin@hefesto.com`) + demo (`demo@hefesto.com`) sin error; el back arranca (boot pasa la validación Joi de DEMO_EMAIL/ADMIN_EMAIL); el front carga el login sin errores en consola.
result: pass

### 2. App name = "Hefesto" (no "Nemea")
expected: Con defaults (sin setear APP_NAME), el título de la pestaña del navegador dice "Hefesto", y el `alt` del logo en el login y en el sidebar es "Hefesto". En ningún texto visible aparece "Nemea".
result: pass

### 3. Acceso-denegado usa branding + email de contacto configurable
expected: Entrando a una ruta sin permiso, la página de acceso denegado muestra el heading/texto con "Hefesto" y el contacto apunta al email configurable (default `admin@hefesto.com`). NO aparece el email personal del autor (basualdofelipe@gmail.com).
result: pass
note: Contacto sale de NEXT_PUBLIC_CONTACT_EMAIL (front), separado del ADMIN_EMAIL del back (que solo siembra el user admin). Mostró el default neutro admin@hefesto.com, sin leak del email personal — correcto por diseño (R10).

### 4. Demo login funciona
expected: En el login, "Entrar como demo" inicia sesión como el usuario demo y el sidebar se puebla (permisos) sin hard reload.
result: pass

### 5. Demo login falla con feedback (toast) — fix WR-01
expected: Con el demo login mal configurado o el back caído, clickear "Entrar como demo" muestra un toast de error ("No se pudo iniciar el login demo...") en vez de un dead-end silencioso (el spinner para y aparece el toast).
result: pass

### 6. Swagger usa APP_NAME
expected: La UI de Swagger del back muestra el título "Hefesto API" (y la descripción con "Hefesto"), no "Nemea".
result: pass

### 7. White-label override (rebranding)
expected: Seteando `NEXT_PUBLIC_APP_NAME=Foo` (front) + `APP_NAME=Foo` (back) y rebuildeando el front, el título/alt/Swagger muestran "Foo". (La parte de seed-from-env con override ya fue verificada empíricamente contra DB fresca en esta sesión: APP_NAME=Foo → demo@foo.com/"Demo Foo".)
result: pass
note: Seed-from-env override verificado empíricamente esta sesión (APP_NAME=Foo → demo@foo.com/"Demo Foo"; ADMIN_EMAIL=x@y.com → x@y.com/"Bar"). Override del front apoyado en el mismo mecanismo de lectura que el default (Test 2 mostró "Hefesto").
