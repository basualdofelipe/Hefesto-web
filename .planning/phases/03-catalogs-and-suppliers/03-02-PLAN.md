```typescript
import { Column, Entity, Index } from 'typeorm';
import { BaseEntity } from '../../common/entities/base.entity';

@Entity('suppliers')
@Index('idx_suppliers_name_active', ['name'], { where: '"is_active" = true', unique: true })
export class Supplier extends BaseEntity {
  @Column({ type: 'varchar', length: 255, nullable: false })
  name!: string;

  @Column({ type: 'varchar', length: 500, nullable: true })
  address!: string | null;

  @Column({ type: 'varchar', length: 255, nullable: true })
  email!: string | null;

  @Column({ type: 'varchar', length: 50, nullable: true })
  phone!: string | null;

  @Column({ type: 'varchar', length: 50, nullable: true })
  whatsapp!: string | null;

  @Column({ type: 'text', nullable: true })
  description!: string | null;

  @Column({ name: 'is_active', type: 'boolean', default: true })
  isActive!: boolean;
}
```

**IMPORTANT:** Do NOT use `@Unique(['name'])` on the supplier entity. The partial unique index `idx_suppliers_name_active` handles uniqueness only for active suppliers (per Pitfall 6 in research). This allows reactivating a supplier with the same name as a deactivated one.

**Step 2: Create CatalogsService** with a generic approach using a dimension-to-repository map:

- Inject all 6 repositories via `@InjectRepository(Entity)`
- Create a private `getRepository(dimension: string)` method that maps dimension strings to repositories:
  - `'product-types'` → `productTypeRepo`
  - `'product-names'` → `productNameRepo`
  - `'product-finishes'` → `productFinishRepo`
  - `'product-colors'` → `productColorRepo`
  - `'product-sizes'` → `productSizeRepo`
  - `'supply-types'` → `supplyTypeRepo`
- Throw `NotFoundException('Dimension "${dimension}" no encontrada')` for invalid dimensions.

Methods:
- `findAll(dimension: string)` → `repo.find({ order: { name: 'ASC' } })`
- `findOne(dimension: string, id: string)` → `repo.findOne({ where: { id } })`, throw `NotFoundException('Item no encontrado')` if null
- `create(dimension: string, dto: CreateCatalogItemDto)` → `repo.create(dto)` then `repo.save()`. Catch unique constraint violation and throw `ConflictException('Ya existe un item con ese nombre')`
- `update(dimension: string, id: string, dto: UpdateCatalogItemDto)` → findOne + merge + save. Catch unique constraint on name
- `remove(dimension: string, id: string)` → findOne first, then attempt `repo.remove()`. **CRITICAL for future FK safety:** Wrap the remove in try/catch. If a QueryFailedError with code `23503` (foreign key violation) is thrown, catch it and throw `ConflictException('No se puede eliminar: este item esta siendo usado por otros registros')`. In Phase 3 there are no FK references yet, but this check prepares for Phase 5 when products reference catalog dimensions.

Also expose a `getValidDimensions()` method returning the list of valid dimension strings (useful for Swagger docs and validation).

**Step 3: Create SuppliersService**:

Inject `@InjectRepository(Supplier)`. Methods:

- `findAll()` → `repo.find({ order: { name: 'ASC' } })` — returns ALL suppliers (including inactive — the frontend handles dimming)
- `findActive()` → `repo.find({ where: { isActive: true }, order: { name: 'ASC' } })` — for future use by other modules (e.g., when selecting a supplier for a supply)
- `findOne(id: string)` → `repo.findOne({ where: { id } })`, throw NotFoundException if null
- `create(dto: CreateSupplierDto)` → `repo.create(dto)` then `repo.save()`. Catch unique constraint (partial index) and throw ConflictException('Ya existe un proveedor activo con ese nombre')
- `update(id: string, dto: UpdateSupplierDto)` → findOne + merge + save. Catch unique constraint
- `toggleStatus(id: string)` → findOne, flip `isActive`, save. Return the updated supplier. This is the soft delete mechanism.
- `remove(id: string)` → Hard delete. Protected by future FK constraints. Wrap in try/catch for FK violation → ConflictException('No se puede eliminar: este proveedor tiene insumos asociados')

**Step 4: Create SuppliersController**:

Endpoints:
- `GET /api/suppliers` — List all suppliers (authenticated, any role). Returns all including inactive.
- `GET /api/suppliers/:id` — Get one supplier (authenticated, any role, ParseUUIDPipe)
- `POST /api/suppliers` — Create supplier (@Roles(Role.ADMIN))
- `PUT /api/suppliers/:id` — Update supplier (@Roles(Role.ADMIN), ParseUUIDPipe)
- `PATCH /api/suppliers/:id/toggle-status` — Toggle active/inactive (@Roles(Role.ADMIN), ParseUUIDPipe). Returns the updated supplier.
- `DELETE /api/suppliers/:id` — Hard delete (@Roles(Role.ADMIN), ParseUUIDPipe)

All endpoints use Swagger decorators: @ApiTags('suppliers'), @ApiBearerAuth, @ApiOperation, @ApiResponse.
