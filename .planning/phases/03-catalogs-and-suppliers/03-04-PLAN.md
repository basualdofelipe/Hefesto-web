<gap_analysis>
**Gap 1 (BLOCKER): Supplier email validation chain is broken**
- Root cause: Three layers of validation conflict when email is left blank.
  1. HTML `type='email'` on the Input triggers browser-native validation, which can block form submission on partial input.
  2. Zod schema uses `.email().optional().or(z.literal(''))` — this allows empty string to pass zod, but the form sends `''` (not `undefined`) to the backend.
  3. Backend `@IsEmail() @IsOptional()` in `CreateSupplierDto` — class-validator's `@IsOptional()` only skips validation for `undefined` or `null`, NOT for empty string `''`. So `''` is treated as an invalid email and rejected with 400.
- Fix strategy: Two-pronged fix.
  A. **SupplierForm.tsx**: Remove `type='email'` from the email Input (eliminates browser-native validation). Change zod schema for email to `z.string().email('Email invalido').or(z.literal('')).optional()` (keeps the validation order correct — empty string passes, non-empty must be valid email).
  B. **nuevo/page.tsx and EditSupplierClient.tsx**: Add a `cleanSupplierData()` helper that transforms empty strings to `undefined` before sending to the backend. This ensures `@IsOptional()` in class-validator correctly skips `@IsEmail()` when no email is provided.

**Gap 2 (BLOCKER): Catalog tabs wrap on mobile instead of scrolling**
- Root cause: `TabsList className='flex-wrap'` in `catalogos/page.tsx` causes the 6 tabs to wrap to multiple lines on narrow viewports.
- Fix: Replace `flex-wrap` with `w-full overflow-x-auto` on TabsList. The TabsTrigger component already has `whitespace-nowrap` in its Shadcn base styles, so horizontal scrolling will work correctly.

**Gap 3 (NEEDS HUMAN): Mobile sidebar backdrop**
- Root cause: Structurally correct — `SheetOverlay` in `sheet.tsx` has `bg-black/50` and is rendered inside `SheetContent`. The Shadcn sidebar uses `Sheet` for mobile, which should render the overlay.
- Fix: No code change needed. This requires human testing on a mobile viewport to confirm the overlay renders and dismisses correctly.
</gap_analysis>

```typescript
// nemea-back/src/suppliers/dto/create-supplier.dto.ts
export class CreateSupplierDto {
  @IsString() @IsNotEmpty() @MaxLength(255) name!: string;
  @IsString() @IsOptional() @MaxLength(500) address?: string;
  @IsEmail({}) @IsOptional() email?: string;       // Skips @IsEmail when value is undefined
  @IsString() @IsOptional() @MaxLength(50) phone?: string;
  @IsString() @IsOptional() @MaxLength(50) whatsapp?: string;
  @IsString() @IsOptional() description?: string;
}
```

Key insight: `@IsOptional()` only skips subsequent validators when the value is `undefined` or `null`. Empty string `''` is NOT skipped — it runs `@IsEmail()` which rejects it. The frontend must send `undefined` (omit the key) for blank optional fields.
