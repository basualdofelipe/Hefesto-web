  1. For each of the 5 tables (product_types, product_names, product_finishes, product_colors, product_sizes):
     - ADD COLUMN sku_code SMALLINT NULL
  2. Seed sku_code values for existing catalog items from the seed migration (1772380800000). Read that migration to find the exact names seeded. Assign sequential sku_code values starting from 1 for each table. For product_sizes, seed "Talle Unico" with sku_code = 0 if not already seeded.

  4. Down migration: drop constraints, drop columns

  1. CREATE TABLE products with columns:
     - id UUID DEFAULT gen_random_uuid() PRIMARY KEY

  <action>
  **DTOs:**
  - CreateProductDto: typeId (UUID), nameId (UUID), finishId (UUID), colorId (UUID), sizeId (UUID). All required, IsUUID.
  - CreateBatchProductsDto: typeId, nameId, finishId (all UUID required), colorIds (UUID[] with ArrayMinSize(1)), sizeIds (UUID[] with ArrayMinSize(1)).
  - UpdateProductDto: same fields as CreateProductDto, all required (SKU regenerates on any change).

  **ProductsService:**
  - Constructor injects: Product repo, and repos for ProductType, ProductName, ProductFinish, ProductColor, ProductSize via TypeOrmModule.forFeature. Also inject EntityManager for transactions.
  - `generateSkuCode(type, name, finish, color, size): string` -- private, returns `${type.skuCode}.${name.skuCode}.${finish.skuCode}.${color.skuCode}.${size.skuCode}`
  - `findAll(includeInactive: boolean): Promise<Product[]>` -- find with where: { isActive: true } unless includeInactive. Order by type.name ASC, then name.name ASC. Do NOT include BOM or prices.
  - `findOne(id: string): Promise<Product>` -- findOne with relations loaded (eager handles it). Throw NotFoundException if not found.
  - `create(dto: CreateProductDto): Promise<Product>` -- load all 5 dimension entities (throw NotFoundException for any missing), generate SKU, create and save product. Catch 23505 for duplicate SKU -> ConflictException("Ya existe un producto activo con esa combinacion").
  - `createBatch(dto: CreateBatchProductsDto): Promise<Product[]>` -- use entityManager.transaction(). Load type, name, finish. Load all colors by IDs and all sizes by IDs. Generate cartesian product (color x size), build Product entities with SKU for each. Save all in transaction. Catch 23505 -> ConflictException with info about which combination(s) already exist.
  - `update(id: string, dto: UpdateProductDto): Promise<Product>` -- load product, load all 5 new dimensions, regenerate SKU, save. Catch 23505 -> ConflictException.
  - `toggleStatus(id: string): Promise<Product>` -- load product, flip isActive, save.

  **ProductsController** (mirroring SuppliesController pattern):
  - @ApiTags('Products'), @ApiBearerAuth(), @Controller('products')
  - GET /products?includeInactive=false -- findAll
  - GET /products/:id -- findOne
  - POST /products -- @Roles(Role.ADMIN), create
  - POST /products/batch -- @Roles(Role.ADMIN), createBatch
  - PUT /products/:id -- @Roles(Role.ADMIN), update
  - PATCH /products/:id/toggle-status -- @Roles(Role.ADMIN), toggleStatus
  - All endpoints with full Swagger decorators (@ApiOperation, @ApiResponse, @ApiQuery where needed)

  **ProductsModule:**
  - TypeOrmModule.forFeature([Product, ProductType, ProductName, ProductFinish, ProductColor, ProductSize])
  - Export ProductsService for future BOM/cost modules

  **AppModule:** Add ProductsModule to imports array.

  Error messages in Spanish (same convention as SuppliesModule).
