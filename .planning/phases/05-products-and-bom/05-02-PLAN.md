  **Entity: SuppliesPerProductHistory** (supplies-per-product-history.entity.ts):
  - Extends BaseEntity
  - ManyToOne to Product (nullable: false, onDelete: 'CASCADE'), JoinColumn 'product_id'
  - ManyToOne to Supply (nullable: false, eager: true), JoinColumn 'supply_id'
  - quantity: decimal(10,2) NOT NULL (typed as string per TypeORM decimal pattern)
  - isActive: boolean, default true
  - Note: NO unit_type column -- unit comes from supply.unitType at read time
  - Same supply CAN appear multiple times per product (no unique constraint on product_id + supply_id)

  **Entity: ProductPriceHistory** (product-price-history.entity.ts):
  - Extends BaseEntity
  - ManyToOne to Product (nullable: false, onDelete: 'CASCADE'), JoinColumn 'product_id'
  - price: decimal(12,2) NOT NULL (typed as string)
  - currency: varchar(10), default 'ARS'

  **Migration 1772500200000-CreateBomAndPriceHistory.ts** (raw SQL):
  1. CREATE TABLE supplies_per_product_history:
     - id UUID DEFAULT gen_random_uuid() PRIMARY KEY
     - product_id UUID NOT NULL REFERENCES products(id) ON DELETE CASCADE
     - supply_id UUID NOT NULL REFERENCES supplies(id)
     - quantity DECIMAL(10,2) NOT NULL
     - is_active BOOLEAN NOT NULL DEFAULT true
     - created_at TIMESTAMPTZ NOT NULL DEFAULT now()
     - updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
  2. CREATE INDEX IDX_bom_product_active ON supplies_per_product_history (product_id) WHERE is_active = true
  3. CREATE TABLE product_price_history:
     - id UUID DEFAULT gen_random_uuid() PRIMARY KEY
     - product_id UUID NOT NULL REFERENCES products(id) ON DELETE CASCADE
     - price DECIMAL(12,2) NOT NULL
     - currency VARCHAR(10) NOT NULL DEFAULT 'ARS'
     - created_at TIMESTAMPTZ NOT NULL DEFAULT now()
     - updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
  4. CREATE INDEX IDX_product_price_created ON product_price_history (product_id, created_at DESC)
  5. Down: DROP TABLE product_price_history, DROP TABLE supplies_per_product_history

  **DTOs:**
  - UpdateBomDto: items array of { supplyId: UUID required, quantity: number required (IsNumber, Min(0.01)) }. Use ValidateNested + Type decorator. The items array CAN be empty (clears the BOM).
  - CreateProductPriceDto: price: number required (IsNumber, Min(0.01)). Swagger example.
  - BatchProductPriceDto: productIds: UUID[] (ArrayMinSize(1)), price: number required (IsNumber, Min(0.01)).

  **ProductsService additions** (add to existing service from 05-01):
  - `getBom(productId: string): Promise<SuppliesPerProductHistory[]>` -- find where product.id = productId AND isActive = true, with supply relation loaded (eager handles it). Verify product exists first.
  - `updateBom(productId: string, dto: UpdateBomDto): Promise<SuppliesPerProductHistory[]>` -- use entityManager.transaction():
    1. Verify product exists
    2. Verify all supply IDs exist and are active (throw NotFoundException for missing, ConflictException for inactive supply)
    3. UPDATE supplies_per_product_history SET is_active = false WHERE product_id = :id AND is_active = true
    4. INSERT new entries with is_active = true for each item in dto.items
    5. Return new active entries
  - `batchUpdateBom(productIds: string[], dto: UpdateBomDto): Promise<void>` -- use transaction, call updateBom logic for each productId. Verify all products exist.
  - `getPriceHistory(productId: string): Promise<ProductPriceHistory[]>` -- find where product.id, order by createdAt DESC. Verify product exists.
  - `addPrice(productId: string, dto: CreateProductPriceDto): Promise<ProductPriceHistory>` -- verify product exists, create and save new record.
  - `batchAddPrice(dto: BatchProductPriceDto): Promise<ProductPriceHistory[]>` -- use transaction, create price record for each productId.

  Also update `findAll` to include current selling price via DISTINCT ON pattern (same as supplies):
  - After loading products, query: `SELECT DISTINCT ON (product_id) * FROM product_price_history ORDER BY product_id, created_at DESC`
  - Map prices to products, return as ProductWithPrice type (Product + currentPrice: number | null + lastPriceUpdate: string | null)

  **ProductsController additions:**
  - GET /products/:id/bom -- getBom
  - PUT /products/:id/bom -- @Roles(ADMIN), updateBom
  - PUT /products/batch-bom -- @Roles(ADMIN), batchUpdateBom
  - GET /products/:id/prices -- getPriceHistory
  - POST /products/:id/prices -- @Roles(ADMIN), addPrice
  - POST /products/batch-prices -- @Roles(ADMIN), batchAddPrice
  - All with full Swagger decorators

  **ProductsModule update:** Add SuppliesPerProductHistory and ProductPriceHistory to TypeOrmModule.forFeature. Also add Supply entity for BOM validation.

  IMPORTANT: The batch-bom and batch-prices routes MUST be defined BEFORE the :id routes in the controller to avoid NestJS treating "batch-bom" as a UUID param.
