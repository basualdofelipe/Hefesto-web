    **types.ts:**
    Create shared types file:
    ```typescript
    export interface ExpenseCategory { id: string; name: string; }
    export interface Expense {
      id: string;
      amount: string; // decimal from backend (parse with parseFloat)
      concept: string;
      date: string; // "YYYY-MM-DD" format from PostgreSQL DATE
      category: ExpenseCategory;
      createdAt: string;
      updatedAt: string;
    }
    ```
    Add CATEGORY_COLORS map: 'materia prima' -> amber, 'packaging' -> blue, 'envio' -> green, 'herramientas' -> purple, 'servicios' -> cyan, 'otros' -> gray. Each with bg-{color}-100 text-{color}-800 Tailwind classes. Add `getCategoryColor(name: string): string` function with lowercase lookup + gray fallback for user-created categories.
    Add `formatAmount(amount: string): string` using parseFloat + toLocaleString('es-AR') for ARS formatting.
    Add `formatMonthLabel(key: string): string` -- takes "YYYY-MM", returns capitalized Spanish month + year (e.g., "Marzo 2026") using `date.toLocaleDateString('es-AR', { month: 'long', year: 'numeric' })`.
    Add `groupByMonth(expenses: Expense[]): Map<string, Expense[]>` -- groups by `expense.date.substring(0, 7)`.
    Add `getCurrentMonthRange(): { dateFrom: string; dateTo: string }` -- returns first and last day of current month in YYYY-MM-DD format.

    **AppSidebar.tsx:**
    Restructure to match user decision. New sidebar order:
    1. Top items (no label): Inicio
    2. "Finanzas" group: Gastos (icon: Receipt from lucide-react, href: /finanzas/gastos)
    3. No label group: Productos (icon: ShoppingBag, href: /productos) -- moved OUT of Datos base
    4. "Datos base" group: Catalogos, Proveedores, Insumos (Productos removed from here)

    Create separate arrays: TOP_ITEMS, FINANZAS_ITEMS, PRODUCTOS_ITEMS, DATOS_BASE_ITEMS. Each renders as its own SidebarGroup with the appropriate SidebarGroupLabel (or no label for top/productos).

    **ExpenseFormDialog.tsx:**
    Client component. Adapt SupplyFormDialog pattern:
    - Props: `expense?: Expense` (undefined for create, filled for edit), `categories: ExpenseCategory[]`, `open: boolean`, `onOpenChange`, `onSuccess: () => void`
    - Zod schema: amount (string, min 1, refine parseFloat > 0), concept (string, min 1, max 500), date (string, min 1), categoryId (string, uuid)
    - react-hook-form with zodResolver
    - On open: reset form. If editing, set defaultValues from expense (amount as string, categoryId from expense.category.id). If creating, default date to today (new Date().toISOString().split('T')[0]).
    - Date field: native `<Input type="date">` (per research recommendation -- funcionalidad primero)
    - Category field: Shadcn Select with categories as options
    - Submit: POST /api/expenses (create) or PUT /api/expenses/:id (edit) via apiClientFetch. Toast success. Call onSuccess.
    - Dialog title: "Nuevo gasto" or "Editar gasto"

    **DeleteExpenseDialog.tsx:**
    Client component. Shadcn AlertDialog pattern:
    - Props: `expense: Expense | null`, `open: boolean`, `onOpenChange`, `onSuccess: () => void`
    - Shows "Eliminar este gasto?" with expense concept and amount in description
    - Confirm button: DELETE /api/expenses/:id via apiClientFetch. Toast success. Call onSuccess.
    - Cancel button closes dialog.

    **ExpenseMonthGroup.tsx:**
    Client component. Adapt SupplyTypeGroup collapsible pattern:
    - Props: `monthKey: string`, `expenses: Expense[]`, `isAdmin: boolean`, `onEdit: (expense: Expense) => void`, `onDelete: (expense: Expense) => void`
    - Collapsible header: formatMonthLabel(monthKey) + expense count + subtotal (sum of parseFloat(amount), formatted with formatAmount)
    - Collapsible content: Table with columns Fecha, Concepto, Categoria, Monto
    - Fecha: display date as-is from string (DO NOT pass through new Date() to avoid timezone shift per Pitfall 2). Format with simple string splitting: "2026-03-06" -> "06/03/2026".
    - Concepto: plain text
    - Categoria: Badge with getCategoryColor(category.name) classes and category.name text
    - Monto: formatAmount(amount) prefixed with "$"
    - If isAdmin: edit (Pencil icon) and delete (Trash2 icon) action buttons per row
    - Default: all month groups open (defaultOpen={true} on Collapsible)

    **ExpenseTable.tsx:**
    Client component. Main orchestrator:
    - Props: `initialExpenses: Expense[]`, `categories: ExpenseCategory[]`, `isAdmin: boolean`
    - State: expenses (from initial), selectedCategory (string | null), dateFrom/dateTo (string, default to getCurrentMonthRange()), editingExpense, deletingExpense, formDialogOpen
    - Filter controls row: category Select (options from categories prop + "Todas" default), two `<Input type="date">` for dateFrom/dateTo
    - When filters change, re-fetch from backend: GET /api/expenses?categoryId=X&dateFrom=Y&dateTo=Z via apiClientFetch. Update expenses state.
    - Summary bar: Card or div showing "Total: $X.XXX" + "N gastos" + filtered period text. Derived via useMemo from current expenses array (not separate calculation -- per Pitfall 4).
    - "+ Nuevo gasto" button (visible only if isAdmin) opens ExpenseFormDialog in create mode
    - Group expenses by month using groupByMonth, render ExpenseMonthGroup for each group
    - onEdit opens ExpenseFormDialog with expense, onDelete opens DeleteExpenseDialog
    - onSuccess callback: re-fetch expenses with current filters, close dialogs
    - Empty state: "No hay gastos en este periodo. Usa el selector de fechas para ver otros meses."

    **page.tsx (finanzas/gastos):**
    Server component at `nemea-front/src/app/(app)/finanzas/gastos/page.tsx`:
    - Uses auth() to get session, determines isAdmin
    - Fetches categories: apiFetch('/api/catalogs/expense-categories')
    - Fetches expenses with current month default: apiFetch(`/api/expenses?dateFrom=YYYY-MM-01&dateTo=YYYY-MM-DD`) using getCurrentMonthRange from types.ts (import and compute on server)
    - Passes data to ExpenseTable client component
    - Page header: "Gastos" title + "Registra y gestiona los gastos del negocio." subtitle

    **catalogos/page.tsx:**
    Add new entry to DIMENSIONS array: `{ key: 'expense-categories', label: 'Categorias de Gasto' }`. This automatically renders the new tab using the existing CatalogTabContent component with inline CRUD. No other changes needed.
