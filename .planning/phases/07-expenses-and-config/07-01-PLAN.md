  <action>
    **Entity:**
    Create `expense.entity.ts` extending BaseEntity with:
    - `amount`: decimal(12,2), not null (TypeORM returns string, same as SupplyPriceHistory.price)
    - `concept`: varchar(500), not null
    - `date`: type 'date' (PostgreSQL DATE, NOT timestamptz -- expenses are calendar dates, avoids timezone shift per Pitfall 2)
    - `category`: ManyToOne to ExpenseCategory, eager: true, nullable: false, JoinColumn name: 'category_id'
    - NO is_active field -- hard delete per user decision

    **DTOs:**
    - `CreateExpenseDto`: amount (IsNumber, Min(0.01), maxDecimalPlaces: 2), concept (IsString, IsNotEmpty), date (IsDateString), categoryId (IsUUID). All with @ApiProperty examples.
    - `UpdateExpenseDto`: PartialType(CreateExpenseDto) -- all fields optional for partial update.
    - `QueryExpensesDto`: categoryId (IsUUID, IsOptional), dateFrom (IsDateString, IsOptional), dateTo (IsDateString, IsOptional). All with @ApiPropertyOptional. Use @IsOptional() before each validator.

    **Service:**
    Create `ExpensesService` with:
    - `findAll(query: QueryExpensesDto): Promise<Expense[]>` -- uses QueryBuilder with leftJoinAndSelect on category, orderBy date DESC + createdAt DESC. Applies optional .andWhere for categoryId, dateFrom (>=), dateTo (<=). Returns flat list (frontend does month grouping).
    - `findOne(id: string): Promise<Expense>` -- findOne by id, throw NotFoundException if not found.
    - `create(dto: CreateExpenseDto): Promise<Expense>` -- look up ExpenseCategory by dto.categoryId (throw NotFoundException if invalid), create expense with category relation, save and return.
    - `update(id: string, dto: UpdateExpenseDto): Promise<Expense>` -- find expense, if dto.categoryId provided look up category, merge and save.
    - `remove(id: string): Promise<void>` -- find expense, then repo.remove (hard delete).

    Inject both `Repository<Expense>` and `Repository<ExpenseCategory>` via @InjectRepository.

    **Controller:**
    Create `ExpensesController` with @ApiTags('expenses'), @ApiBearerAuth():
    - `GET /expenses` -- @Query(ValidationPipe with transform+whitelist) QueryExpensesDto. Public to authenticated users (no @Roles needed, GET is read-only).
    - `POST /expenses` -- @Roles(Role.ADMIN), @Body() CreateExpenseDto
    - `PUT /expenses/:id` -- @Roles(Role.ADMIN), @Param('id', ParseUUIDPipe), @Body() UpdateExpenseDto
    - `DELETE /expenses/:id` -- @Roles(Role.ADMIN), @Param('id', ParseUUIDPipe)

    All endpoints with @ApiOperation and @ApiResponse decorators.

    **Module:**
    Create `ExpensesModule` importing TypeOrmModule.forFeature([Expense, ExpenseCategory]), registering controller and service.

    **AppModule:**
    Add `ExpensesModule` to imports array in app.module.ts.

    **Migration (1772600000000-CreateExpenseCategoriesAndExpenses.ts):**
    Single migration that:
    1. Creates `expense_categories` table (id uuid PK default uuid_generate_v4(), created_at timestamptz, updated_at timestamptz, name varchar(100) UNIQUE, PK constraint named PK_expense_categories_id, unique constraint named UQ_expense_categories_name)
    2. Seeds 6 categories: 'Materia prima', 'Packaging', 'Envio', 'Herramientas', 'Servicios', 'Otros'
    3. Creates `expenses` table (id uuid PK, created_at, updated_at, amount decimal(12,2) NOT NULL, concept varchar(500) NOT NULL, date DATE NOT NULL, category_id uuid NOT NULL FK to expense_categories(id), PK named PK_expenses_id, FK named FK_expenses_category)
    4. Creates index IDX_expenses_date on expenses(date DESC) for filter performance
    5. Down migration drops expenses first, then expense_categories (FK order)

    Use raw SQL in queryRunner.query (same pattern as existing migrations).
