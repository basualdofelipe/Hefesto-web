## 5. Paso acoplado D-04: renombrar carpetas locales + `.gitignore` (JUNTOS, antes de publicar)

**Cuándo:** En el momento final pre-publicación, cuando la sesión de desarrollo
activa ya no dependa de las rutas `nemea-front`/`nemea-back`.

**Por qué están acoplados:** Si se renombra la carpeta sin actualizar `.gitignore`,
el orquestador deja de ignorar el sub-repo renombrado y empieza a trackearlo como
contenido nuevo — colateral no deseado. Los dos pasos deben ejecutarse juntos.

- [ ] Renombrar carpetas locales:
  ```bash
  # desde la raíz del orquestador
  mv nemea-front hefesto-front
  mv nemea-back hefesto-back
  ```

- [ ] Actualizar `.gitignore` del orquestador raíz (en el mismo paso):
  Reemplazar en `.gitignore`:
  ```
  /nemea-front/
  /nemea-back/
  ```
  por:
  ```
  /hefesto-front/
  /hefesto-back/
  ```

- [ ] Verificar que git no trackea el contenido del sub-repo después del rename:
  ```bash
  git status --short
  # Debe mostrar solo el cambio al .gitignore, nada de hefesto-front/* ni hefesto-back/*
  ```

- [ ] Commitear el cambio al `.gitignore`:
  ```bash
  git add .gitignore
  git commit -m "chore: rename gitignore entries nemea-front/back → hefesto-front/back (D-04 coupled step)"
  ```
