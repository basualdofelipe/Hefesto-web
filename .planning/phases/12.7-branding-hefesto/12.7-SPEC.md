## Goal

Ningún repo público del portfolio —`nemea-front`, `nemea-back`, Y el orquestador (que va PÚBLICO como showcase del proceso GSD)— muestra el literal `nemea` (case-insensitive). Todo dice Hefesto: docker, DB, env de ejemplo, paquetes, código, docs y `.planning`.

## Background

"Nemea" es marca registrada de los inversores de la marroquinería; para portfolio se rebrandea a **Hefesto** y se evita el conflicto. La Phase 12.6 ya hizo el branding **in-app** configurable por env con defaults neutros (`APP_NAME=Hefesto`, etc.) — un clon que arranca con `.env.example` ya muestra "Hefesto". Lo que sigue hardcodeado con `nemea` es **plomería**.

**Decisión 2026-06-16:** el repo orquestador (Nemea-web, el del `.planning`) va **PÚBLICO** como showcase de cómo se trabaja con GSD (señal de proceso/dirección de AI). Por eso `.planning/` también entra al rebrand: ~60 archivos con "Nemea" en títulos/prose.

Las migraciones **no** contienen `nemea` (productos seed = dioses griegos: Hefesto, Ares, Hermes…) → la data seed no se toca. Inventario verificado: `~/.claude/notes/Nemea-web/2026-06-15_rebrand-nemea-to-hefesto.md`.

## Requirements

1. **Infra naming (docker + DB)**: docker-compose y env de ejemplo usan `hefesto_*` de forma consistente.
   - Current: `docker-compose.yml` usa `nemea_postgres`/`nemea_db`/user `nemea`/pass `nemea_dev`/volume `nemea_pgdata` (+ `_test`); `.env.example` `DATABASE_URL=postgresql://nemea:nemea_dev@localhost:5432/nemea_db` (+ `DATABASE_URL_TEST`).
   - Target: todos esos tokens → `hefesto_*`; valores de `docker-compose.yml` y `.env.example` mutuamente consistentes.
   - Acceptance: `git grep -i nemea -- docker-compose.yml .env.example` vacío; `docker compose up -d` levanta y conecta a `hefesto_db` sin error.

2. **Package identity (front + back)**: `name` de `package.json` + self-name del lockfile = `hefesto-front`/`hefesto-back`.
   - Current: `"name": "nemea-front"`/`"nemea-back"`; `package-lock.json` repite el self-name (2 refs/repo).
   - Target: `hefesto-front`/`hefesto-back` en `package.json` y `package-lock.json`.
   - Acceptance: `git grep -i nemea -- package.json package-lock.json` vacío en ambos repos; `npm ci` corre.

3. **Code references (back)**: health response del controller + contexto del logger.
   - Current: `app.controller.ts` devuelve `app: 'nemea-back'`; `logging.interceptor.ts` usa `new Logger('Nemea')`.
   - Target: `'hefesto-back'` y `new Logger('Hefesto')`.
   - Acceptance: `GET /` devuelve `app: "hefesto-back"`; `git grep -i nemea -- src` vacío en back.

4. **Docs user-facing (front + back + root)**: READMEs, ARCHITECTURE y docs del orquestador sin `nemea`.
   - Current: README + `docs/ARCHITECTURE.md` en front y back; `CLAUDE.md`/`README.md`/`README.es.md` del root referencian `Nemea`/`Nemea-web`.
   - Target: refs de marca → `Hefesto`/`Hefesto-web`. Los tokens de path `nemea-front`/`nemea-back` y el `.gitignore` se resuelven según la decisión de consistencia de carpetas (ver Constraints) — con orquestador público probablemente unificar a `hefesto-*`.
   - Acceptance: `git grep -i nemea -- README.md docs/ARCHITECTURE.md` vacío en front y back; root `git grep -n "Nemea" -- CLAUDE.md README.md README.es.md` vacío.

5. **Test fixtures**: specs/e2e rebrandeados para que la suite quede verde y grep-clean.
   - Current: 4 test files de front + 8 specs/e2e de back contienen `nemea`.
   - Target: fixtures usan hefesto; fuente y fixtures cambian juntas.
   - Acceptance: `npm test` (unit) verde en ambos repos; `git grep -i nemea` no encuentra fixtures con nemea.

6. **Mapping sin colateral**: solo cambian tokens de marca; migraciones y data seed intactas.
   - Current: las migraciones contienen cero `nemea`; mapping case-sensitive `nemea→hefesto`, `Nemea→Hefesto`, `NEMEA→HEFESTO`.
   - Target: `src/database/migrations/**` sin cambios; ninguna palabra con `nemea` como substring queda mal reemplazada.
   - Acceptance: `git diff --stat` muestra cero cambios bajo `src/database/migrations/`; diff-review confirma sin substrings rotos.

7. **Orchestrator `.planning/` rebrand**: el orquestador va público → sus docs no muestran "Nemea".
   - Current: ~60 archivos en `.planning/` (PROJECT.md, REQUIREMENTS.md, ROADMAP.md, `codebase/*`, y PLAN/SUMMARY/CONTEXT/RESEARCH de casi todas las fases) contienen "Nemea" en títulos/prose. No hay tool GSD de rebrand; es find-replace.
   - Target: `Nemea`/`nemea` → `Hefesto`/`hefesto` en `.planning/`. Los archivos que DOCUMENTAN este rebrand (este SPEC, notas/summaries de 12.7) se **reescriben a framing neutro** ("el proyecto se rebrandeó a Hefesto"), NO se blind-sed-ean — no deben quedar incoherentes ni airear el tema marca/inversores en repo público.
   - Acceptance: `git grep -i nemea -- .planning` devuelve cero; revisión manual confirma docs meta coherentes y SIN mención del conflicto de marca/inversores.

## Boundaries

**In scope:**
- Reemplazo case-sensitive (`nemea→hefesto`, `Nemea→Hefesto`, `NEMEA→HEFESTO`) en `nemea-front` y `nemea-back`: código, docker-compose, `.env.example`, `package.json` + lockfile, docs, fixtures de test.
- Repo orquestador (root): docs user-facing (`CLAUDE.md`, READMEs) **y `.planning/`** (~60 archivos) — va público como showcase del proceso GSD.
- Checklist de follow-up manual con las acciones de GitHub.

**Out of scope:**
- Renames de repos en GitHub (`nemea-front`→`hefesto-front`, `nemea-back`→`hefesto-back`, `Nemea-web`→`Hefesto-web`), `git remote set-url`, push, About — outward/manual; GSD no pushea ni renombra sin OK. Checklist de follow-up.
- Branding in-app (`APP_NAME`, mails) — ya hecho en Phase 12.6.
- Git history y mensajes de commit — decisión del usuario: no es un riesgo (no relitigar), incluso con el repo público.

## Constraints

- Mapping case-sensitive, 3 reglas; `nemea` no es substring de palabra real → token-replace seguro, pero el executor DEBE diff-review para descartar colateral.
- `docker-compose.yml` consistente con `DATABASE_URL` de `.env.example` (ambos `hefesto`).
- Volume `nemea_pgdata` → `hefesto_pgdata` orphanea data dev; re-seed con `docker compose down -v && up` + migraciones.
- e2e usan la DB de test → recrear container `postgres-test` (hefesto_test) antes de `npm run test:e2e`.
- **Meta-docs (resolver en plan/execute):** los archivos que documentan el rebrand (este SPEC, notas, summaries de 12.7) referencian "Nemea" a propósito. Para repo público se reescriben a framing neutro — NO blind-sed (evita incoherencia + evita airear el tema marca/inversores, mismo autogol que poner un cartel "originalmente Nemea").
- **Consistencia de carpetas (decisión de plan):** con orquestador público, los tokens de path `nemea-front`/`nemea-back` (en docs + `.gitignore`) quedan inconsistentes con los repos GitHub renombrados. Probablemente unificar a `hefesto-*` (puede implicar renombrar las carpetas locales). Resolver en plan.
- `project_title` que aparece en `init` es DERIVADO (no es key de config) — sigue a PROJECT.md / nombre del repo una vez renombrados; no hay setting que tocar.
