## Deviations from Plan

### Auto-handled edge cases

**1. ROADMAP.md Phase 12.7 goal description — circular literal**

- **Found during:** Task 2, ROADMAP.md sweep
- **Issue:** The Phase 12.7 goal description contained the sentence "muestra el literal `nemea` (case-insensitive)" — a code-span reference to the search target itself. Replacing `nemea`→`hefesto` would produce "muestra el literal `hefesto` (case-insensitive)" which is incoherent (the goal would read "no public repo shows `hefesto`").
- **Fix:** Rephrased to "muestra el nombre de marca anterior (case-insensitive)" — preserves the meaning without the literal token.
- **Files modified:** `.planning/ROADMAP.md`
- **Justification:** Diff-review catch per plan instructions: "diff-review every hunk for collateral".

**2. Phase 12.6 goal description — hardcoded string reference**

- **Found during:** Task 2, ROADMAP.md sweep
- **Issue:** Phase 12.6 goal text referred to `("Nemea")` as the hardcoded string that was being replaced by env vars. After rebrand, updating it to `("Hefesto")` correctly describes what is now the default fallback value.
- **Fix:** `("Nemea")`→`("Hefesto")` — semantically correct post-rebrand.
- **Files modified:** `.planning/ROADMAP.md`
