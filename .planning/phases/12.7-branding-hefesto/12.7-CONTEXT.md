## Implementation Decisions

### Identidad real (sanitización más allá de "Nemea")
- **D-01:** Neutralizar la identidad real expuesta en docs (nombres de entidad legal, etc.) lo maneja el **usuario a mano**, FUERA del scope documentado de la fase. NO se automatiza, NO se enumera como checklist en ningún artefacto (el executor solo toca el string `nemea`).

### Scope del rebrand en `.planning/`
- **D-02:** El rebrand de `.planning/` se limita a los docs **principales de raíz**: `PROJECT.md`, `REQUIREMENTS.md`, `ROADMAP.md`, `STATE.md`. **NO** se barre `.planning/phases/*`. Esto **acota el req 7 del SPEC** (que pedía full `.planning`, ~60 archivos). `codebase/*.md`: el planner decide si cuenta como "principal".
- **D-03:** Tradeoff aceptado por el usuario: los artefactos de `phases/` en el repo público conservan "Nemea" (no se limpian). Se prioriza limpiar la "cara" del planning, no el detalle de proceso profundo.

### Carpetas locales y `.gitignore`
- **D-04:** DEFERRED al **paso final** (justo antes de publicar): renombrar carpetas locales `nemea-front`/`nemea-back` → `hefesto-*` Y actualizar `.gitignore`, ejecutados **JUNTOS** (están acoplados: si `.gitignore` dice `hefesto-front/` pero la carpeta sigue `nemea-front/`, git deja de ignorarla y la trackea). Por ahora el local queda `nemea-*` para NO romper la sesión activa.

### Exclusión de la fase 12.7
- **D-05:** Los artefactos de la fase **12.7** (SPEC, CONTEXT, SUMMARY, etc.) se **EXCLUYEN del repo público**. 12.7 queda privada → sus docs pueden nombrar Nemea/rebrand libremente. Esto **reemplaza** el constraint del SPEC de "reescribir meta-docs a neutro" — no hace falta rephrase, simplemente no van al público.

### Claude's Discretion
- Mecánica del sweep (`git grep -li nemea | sed`, orden, per-repo), diff-review como gate anti-colateral, y la verificación final (grep vacío + `npm test` + `docker up`) las resuelve el planner/executor.

</decisions>
