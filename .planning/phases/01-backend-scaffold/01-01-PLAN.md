<task type="auto">
  <name>Task 1: Scaffold NestJS 11 project and install all dependencies</name>
  <files>
    nemea-back/package.json
    nemea-back/tsconfig.json
    nemea-back/tsconfig.build.json
    nemea-back/nest-cli.json
    nemea-back/src/main.ts
    nemea-back/src/app.module.ts
    nemea-back/src/app.controller.ts
    nemea-back/src/app.service.ts
    nemea-back/src/app.controller.spec.ts
  </files>
  <action>
    1. Run `nest new nemea-back --strict --package-manager npm` from the Nemea-web root to scaffold the project. Since nemea-back/ already exists with a .git folder and README, either:
       - Remove the existing nemea-back/ contents first (preserve .git/), scaffold into a temp dir, and move files in, OR
       - Scaffold into a temp name and move files into nemea-back/
       The key: preserve the existing .git/ directory inside nemea-back/.

    2. After scaffold, install ALL production dependencies:
       ```bash
       cd nemea-back
       npm install @nestjs/typeorm typeorm pg
       npm install @nestjs/config joi
       npm install class-validator class-transformer
       npm install helmet @nestjs/throttler
       npm install @nestjs/swagger
       ```

    3. Install ALL dev dependencies:
       ```bash
       npm install -D @swc/cli @swc/core
       npm install -D @nestjs/testing supertest @types/supertest
       npm install -D eslint typescript-eslint eslint-plugin-sonarjs prettier
       npm install -D husky lint-staged
       npm install -D ts-node tsconfig-paths
       ```

    4. Configure SWC builder in nest-cli.json:
       ```json
       {
         "$schema": "https://json-schema.org/nest-cli",
         "collection": "@nestjs/schematics",
         "sourceRoot": "src",
         "compilerOptions": {
           "builder": "swc",
           "typeCheck": true
         }
       }
       ```

    5. Update tsconfig.json to enable strict mode with path aliases:
       - `"strict": true` (should already be set from --strict flag)
       - `"noImplicitAny": true`
       - `"strictBindCallApply": true`
       - `"forceConsistentCasingInFileNames": true`
       - `"noFallthroughCasesInSwitch": true`
       - `"paths": { "@src/*": ["src/*"] }`
       - `"target": "ES2021"`
       - Keep emitDecoratorMetadata and experimentalDecorators (required by NestJS)

    6. Verify the scaffold boots: `npm run start:dev` should start without errors (Ctrl+C after confirming).

    IMPORTANT: Do NOT use `@nestjs/class-validator` or `@nestjs/class-transformer` (unmaintained 4-year-old forks). Use standalone `class-validator` and `class-transformer`.
    IMPORTANT: SWC is incompatible with NestJS CLI plugins (like Swagger CLI plugin). This is fine -- Swagger decorators will be added manually to DTOs in future phases.
  </action>
  <verify>
    <automated>cd nemea-back && npm run build && node dist/main.js &amp; sleep 3 && curl -s http://localhost:3000 && kill %1 2>/dev/null; echo "Build and boot OK"</automated>
  </verify>
  <done>
    - NestJS 11 project exists in nemea-back/ with package.json listing all dependencies
    - `npm run build` succeeds without errors
    - `npm run start:dev` boots the app (default NestJS hello world response)
    - SWC builder configured in nest-cli.json
    - TypeScript strict mode enabled with path aliases
  </done>
</task>

<task type="auto">
  <name>Task 2: Configure ESLint flat config, Prettier, and Husky + lint-staged</name>
  <files>
    nemea-back/eslint.config.mjs
    nemea-back/.prettierrc.json
    nemea-back/.lintstagedrc.json
    nemea-back/.husky/pre-commit
  </files>
  <action>
    1. DELETE the scaffolded `.eslintrc.js` (legacy format generated by `nest new`). Also delete any `.eslintrc.cjs` or `.eslintrc.json` if present.

    2. Create `eslint.config.mjs` matching frontend conventions (ESLint 9 flat config):
       ```javascript
       import eslint from '@eslint/js';
       import tseslint from 'typescript-eslint';
       import sonarjs from 'eslint-plugin-sonarjs';

       export default tseslint.config(
         eslint.configs.recommended,
         ...tseslint.configs.recommended,
         {
           plugins: { sonarjs },
           rules: {
             'sonarjs/cognitive-complexity': ['error', 30],
             'no-debugger': 'error',
             quotes: [
               'error',
               'single',
               { allowTemplateLiterals: true, avoidEscape: true },
             ],
           },
         },
         {
           files: ['**/*.ts'],
           rules: {
             '@typescript-eslint/no-unused-vars': [
               'error',
               {
                 args: 'after-used',
                 argsIgnorePattern: '^_',
                 varsIgnorePattern: '^_',
                 caughtErrors: 'none',
               },
             ],
             '@typescript-eslint/explicit-function-return-type': [
               'error',
               {
                 allowTypedFunctionExpressions: true,
                 allowHigherOrderFunctions: true,
                 allowDirectConstAssertionInArrowFunctions: true,
                 allowExpressions: true,
               },
             ],
             '@typescript-eslint/no-explicit-any': 'error',
           },
         },
         {
           ignores: ['node_modules/**', 'dist/**', 'coverage/**'],
         },
       );
       ```

    3. Create `.prettierrc.json` matching frontend (minus Tailwind plugin and jsxSingleQuote):
       ```json
       {
         "endOfLine": "auto",
         "singleQuote": true,
         "trailingComma": "all",
         "tabWidth": 2,
         "semi": true
       }
       ```

    4. Create `.lintstagedrc.json`:
       ```json
       {
         "*.ts": ["eslint --fix", "prettier --check"],
         "*.json": ["prettier --check"]
       }
       ```

    5. Add npm scripts to package.json if not already present:
       - `"lint": "eslint \"{src,apps,libs,test}/**/*.ts\""`
       - `"prettier": "prettier ."`
       - `"prettier:fix": "prettier --write ."`
       - `"prettier:check": "prettier --check ."`

    6. Initialize Husky: `npx husky init` (creates .husky/ directory)

    7. Create `.husky/pre-commit`:
       ```bash
       npx lint-staged
       ```

    8. Run `npm run lint` to verify ESLint works on the scaffolded code. Fix any lint errors in the scaffolded files (app.controller.ts, app.service.ts, main.ts, app.module.ts) to comply with the new rules -- primarily adding explicit return types.

    9. Run `npm run prettier:fix` to format all files.

    NOTE: If the scaffolded code has ESLint errors (e.g., missing explicit return types on NestJS-generated code), fix them. The scaffolded app.controller.ts likely needs a return type on getHello().
  </action>
  <verify>
    <automated>cd nemea-back && npm run lint && npm run prettier:check && echo "Lint and Prettier OK"</automated>
  </verify>

    - Prettier config matches frontend (single quotes, 2 spaces, trailing commas, auto line ending)
    - lint-staged runs ESLint --fix + Prettier --check on staged .ts files
    - Husky pre-commit hook invokes lint-staged
    - `npm run lint` passes with zero errors on all src/ files
    - `npm run prettier:check` passes on all files
    - No legacy .eslintrc.* files remain
  </done>
