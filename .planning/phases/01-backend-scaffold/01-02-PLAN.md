       NODE_ENV=development
       PORT=4000

       # Database (Docker Compose local)

    3. Copy `.env.example` to `.env` (gitignored). Verify .gitignore includes `.env` but NOT `.env.example`.

    4. Create `src/database/data-source.ts` -- the SINGLE SOURCE OF TRUTH for TypeORM config:
       - Import `dotenv/config` (or use `config()` from dotenv) so CLI usage loads .env
       - Export `dataSourceOptions` with:
         - type: 'postgres'
         - url: process.env.DATABASE_URL
         - ssl: `process.env.NODE_ENV === 'production' ? { rejectUnauthorized: false } : false`
         - entities: `[join(__dirname, '..', '**', '*.entity.{ts,js}')]`
         - migrations: `[join(__dirname, 'migrations', '*.{ts,js}')]`
         - `synchronize: false` (HARDCODED, NEVER conditioned on NODE_ENV)
         - `migrationsRun: true` (auto-run migrations on app start)
         - logging: `process.env.NODE_ENV !== 'production'`
       - Export `AppDataSource = new DataSource(dataSourceOptions)` for CLI usage

       CRITICAL: Do NOT use `autoLoadEntities`. The shared entities array is the contract between CLI and runtime.

    5. Create `src/config/typeorm.config.ts`:
       - Import `dataSourceOptions` from `../database/data-source`
       - Export `typeOrmConfig: TypeOrmModuleOptions` spreading `dataSourceOptions`
       - Do NOT add `autoLoadEntities`

    6. Create empty `src/database/migrations/.gitkeep`

    7. Add migration scripts to package.json (Windows-compatible -- NO `$npm_config_name`):
       ```json
       "typeorm": "npx ts-node -r tsconfig-paths/register ./node_modules/typeorm/cli",
       "migration:generate": "npm run typeorm -- migration:generate -d src/database/data-source",
       "migration:run": "npm run typeorm -- migration:run -d src/database/data-source",
       "migration:revert": "npm run typeorm -- migration:revert -d src/database/data-source"
       ```
       Usage for generate: `npm run migration:generate -- src/database/migrations/CreateUsersTable`

    IMPORTANT: The `migration:generate` script does NOT include a migration path -- the user passes it as a trailing argument. This avoids the Windows `$npm_config_name` pitfall entirely.
    IMPORTANT: `synchronize: false` is UNCONDITIONAL. Not `process.env.NODE_ENV !== 'production'`. Just `false`.

    1. Create `src/common/entities/base.entity.ts` -- abstract base entity:
       ```typescript
       import {
         PrimaryGeneratedColumn,
         CreateDateColumn,
         UpdateDateColumn,
       } from 'typeorm';

       export abstract class BaseEntity {
         @PrimaryGeneratedColumn()
         id: number;

         @CreateDateColumn({ name: 'created_at', type: 'timestamptz' })
         createdAt: Date;

         @UpdateDateColumn({ name: 'updated_at', type: 'timestamptz' })
         updatedAt: Date;
       }
       ```
       This satisfies INFR-04: all tables will have created_at and updated_at timestamps by extending this class.

    2. Create `src/common/types/role.enum.ts` (needed in Phase 2, establish now):

       import { AppController } from './app.controller';
