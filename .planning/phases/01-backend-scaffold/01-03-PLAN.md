  <action>
    1. Create `src/config/env.validation.ts` with Joi schema:
       - NODE_ENV: string, valid('development', 'production', 'test'), default 'development'
       - PORT: number, default 4000
       - DATABASE_URL: string, required
       - FRONTEND_URL: string, required
       App MUST crash with clear error listing ALL missing vars (abortEarly: false).

    2. Create `src/common/filters/http-exception.filter.ts`:
       - Catch all exceptions (use @Catch() with no arguments)
       - HttpException: extract status and response
       - Unknown exceptions: return 500 Internal Server Error
       - Response format: `{ statusCode, error, message, timestamp }`
       - For validation errors (class-validator), `message` should be the array of field error strings (NestJS ValidationPipe returns these as an array)

    3. Create `src/common/interceptors/response.interceptor.ts` (Claude's discretion: using envelope):
       - Wrap all successful responses in `{ data: T, meta?: { total, page, limit } }`
       - For now, meta is not populated (future phases add pagination)
       - Do NOT wrap error responses (the HttpExceptionFilter handles those)
       - Exclude Swagger routes from envelope wrapping (they need raw responses)
       - The health endpoint response will be: `{ data: { status, app, version, uptime } }`

    4. Create `src/common/interceptors/logging.interceptor.ts` (Claude's discretion: using request logging):
       - Log each request: method, URL, status code, response time in ms
       - Use NestJS built-in Logger (not Winston/Pino)
       - Example output: `[Nemea] GET /api/health 200 3ms`

    5. Update `src/app.module.ts` to add:
       - `ConfigModule.forRoot({ isGlobal: true, validationSchema: envValidationSchema, validationOptions: { abortEarly: false } })`
       - `ThrottlerModule.forRoot({ throttlers: [{ ttl: 60000, limit: 100 }] })`
       - `{ provide: APP_GUARD, useClass: ThrottlerGuard }` in providers

    6. Rewrite `src/main.ts` with complete bootstrap:
       - `app.setGlobalPrefix('api')`
       - `app.use(helmet())` -- default config
       - `app.enableCors({ origin: configService.get('FRONTEND_URL'), credentials: true })`
       - `app.useGlobalPipes(new ValidationPipe({ whitelist: true, forbidNonWhitelisted: true, transform: true }))`
       - `app.useGlobalFilters(new HttpExceptionFilter())`
       - `app.useGlobalInterceptors(new ResponseInterceptor(), new LoggingInterceptor())`
       - Swagger setup (dev only): `if (process.env.NODE_ENV !== 'production') { ... }`
         - Title: 'Nemea API'
         - Description: 'Nemea leather goods cost management API'
         - Version: '0.1.0'
         - Setup at path 'api/docs'
       - Listen on `configService.get('PORT') || 4000`
       - Use `ConfigService` from the app container to get env vars (NOT process.env directly in main.ts for PORT and FRONTEND_URL)

    7. Update `src/app.controller.ts` to be the health check endpoint:
       - Route: `GET /health` (becomes `/api/health` with global prefix)
       - Swagger decorators: @ApiTags('health'), @ApiOperation, @ApiResponse
       - Returns: `{ status: 'ok', app: 'nemea-back', version: '0.1.0', uptime: process.uptime() }`
       - Explicit return type interface

    8. Update `src/app.service.ts` if needed (can keep health logic in controller since it's trivial, or delegate to service per NestJS convention -- use your judgment).

    9. Run lint and prettier on all modified files. Fix any issues.

    IMPORTANT: Swagger decorators must be added MANUALLY to DTOs/controllers (SWC is incompatible with the Swagger CLI plugin).
    IMPORTANT: Use `ConfigService` from DI for env access in main.ts. Get it via `app.get(ConfigService)` after creating the app.
