## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`), table cell padding |
| md | 16px | Default element spacing (`gap-4`, `p-4`), card padding |
| lg | 24px | Section padding (`space-y-6`, `p-6`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Empty state vertical padding (`py-12`) |
| 3xl | 64px | Not used in this phase |

Exceptions: none

**Source:** Existing codebase patterns. Pages use `space-y-6` for page-level flow, Cards use `pt-6` / `p-4` / `p-3` for internal spacing.

## Color

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `--background` (oklch 0.99 warm neutral) | Page background, table background |

### Semantic Colors for Financial Data

| Semantic | Light Mode | Dark Mode | Usage |
|----------|-----------|-----------|-------|
| Positive / gain | `text-green-600` | `text-green-400` | Positive margin difference, margin improvement |
| Negative / loss | `text-red-600` | `text-red-400` | Negative margin difference, margin decrease |
| Override highlight | `text-blue-700` on `bg-blue-50` | `text-blue-300` on `bg-blue-950/30` | Override price (to distinguish from real price) |
| Muted | `--muted-foreground` | `--muted-foreground` | Subtitle text, empty state, placeholders, secondary info |
| Inactive product | `--muted-foreground` + `opacity-60` | same | Deactivated products in override table |

**Accent reserved for:**
- "Crear escenario" primary CTA button
- "Guardar y calcular" primary action button in editor
- Scenario name links in the list view
- Active sidebar link highlight

**Source:** DesglosePanel.tsx uses `text-red-600 dark:text-red-400` for negative, `text-green-600 dark:text-green-400` for positive, `text-blue-700 dark:text-blue-300` for highlight-blue. This phase continues that exact pattern.

## Component Inventory

### Page 1: Scenario List (`/escenarios`)

**Primary visual anchor:** Scenario name links on each ScenarioCard -- the clickable name draws the eye and serves as the entry point into each scenario.

| Component | Shadcn Base | Purpose |
|-----------|-------------|---------|
| Page header | None (raw h1 + p) | "Escenarios" title + subtitle |
| ScenarioListClient | None (client wrapper) | State management for list CRUD |
| ScenarioCard | Card + Badge + Button + Tooltip | One card per scenario showing name, date, product count, owner badge for shared |
| Create scenario dialog | Dialog + Input + Select | Modal to create new scenario with name + optional gateway/plan |
| Delete confirmation | AlertDialog | Confirm scenario deletion |
| Empty state | None (div) | When user has zero scenarios |

**ScenarioCard layout:**

```
+-------------------------------------------------------+
| [Scenario Name]                  [Compartido] B  [T]   |
| Creado 28/03/2026  |  12 productos  |  Pago Nube       |
|                                                        |
| [Editar escenario]  [Compartir toggle]                 |
+-------------------------------------------------------+

[T] = Trash2 icon button (size="icon" variant="ghost") with Tooltip "Eliminar escenario"
     aria-label="Eliminar escenario [name]"
```

- Badge "(compartido por Felipe)" shown on other users' public scenarios, read-only (no edit/delete buttons)
- Badge "Publico" shown on own public scenarios

### Page 2: Scenario Editor (`/escenarios/[id]`)

**Primary visual anchor:** ProductOverrideTable -- the full-width table with editable override prices and color-coded margin comparisons is the central workspace of this page.

| Component | Shadcn Base | Purpose |
|-----------|-------------|---------|
| Page header | None | Scenario name as h1 + back link |
| GatewayPlanSelector | Card + Select | Pasarela + plan + payment method override for the scenario |
| BulkOverrideDialog | Dialog + Select + Input | "+X% to type Y" bulk action |
| ProductOverrideTable | Table + Input | Full product table with editable override prices and margin comparison |
| MarginSummary | Card | Aggregate summary: avg margin, total products, products with overrides |
| Save + Calculate button | Button | Primary action to persist overrides and trigger calculation |

**ProductOverrideTable columns:**

| Column | Width | Content |
|--------|-------|---------|
| Producto | flex | Type + Name + Finish + Color (via getProductDisplayName) |
| Costo | 100px | Real cost from DB, right-aligned, `tabular-nums` |
| Precio real | 110px | Current selling price, right-aligned, `tabular-nums` |
| Precio override | 130px | Editable Input with `$` prefix, right-aligned, `font-mono tabular-nums`. Blue text when overridden. |
| Margen real | 100px | `$X (Y%)`, right-aligned, `tabular-nums` |
| Margen simulado | 110px | `$X (Y%)`, green if better than real, red if worse, right-aligned |
| Diferencia | 90px | `+$X` green or `-$X` red with arrow icon (TrendingUp / TrendingDown) |

**Deactivated product rows:** Full row at `opacity-60` with Badge "(inactivo)" after product name. Override field still editable (override preserved per CONTEXT.md).

## Interaction Patterns

### Scenario List Interactions

| Action | Trigger | Behavior |
|--------|---------|----------|
| Create scenario | Click "Crear escenario" button | Opens Dialog. Fields: name (required, text input), gateway (Select, optional), plan (Select, optional). On submit: POST /scenarios, toast success, add card to list. |
| Edit scenario | Click "Editar escenario" on card | Navigate to /escenarios/[id] (scenario editor page) |
| Delete scenario | Click Trash2 icon button on card (tooltip: "Eliminar escenario") | Opens AlertDialog: "Eliminar escenario [name]? Esta accion no se puede deshacer." Confirm: DELETE /scenarios/:id, toast success, remove card. |
| Toggle public | Click Switch on own card | PATCH /scenarios/:id/toggle-public. Toast: "Escenario compartido con inversores" or "Escenario ahora es privado". |

### Scenario Editor Interactions

| Action | Trigger | Behavior |
|--------|---------|----------|
| Edit individual override | Type in override price Input | Updates local state immediately. Blue text indicates an override exists. No API call yet. |
| Clear individual override | Clear the override Input field | Removes override from local state. Cell reverts to non-highlighted state. |
| Bulk override | Click "Ajuste masivo" button | Opens BulkOverrideDialog. Fields: tipo (Select: product types + "Todos"), porcentaje (number Input, supports negative for discount). Preview text: "Aplicar +10% a 8 Cinturones". On confirm: frontend calculates new prices, updates local state, closes dialog, toast: "Override aplicado a 8 productos". |
| Clear all overrides | Click "Limpiar overrides" button | Confirmation toast or inline confirm. Clears all overrides from local state. |
| Save and calculate | Click "Guardar y calcular" button | PUT /scenarios/:id/overrides with all overrides. Then GET /scenarios/:id/calculate. Show loading skeleton on table margins. On complete: update margin columns with real results. Toast: "Escenario calculado". |
| Change gateway/plan | Change any Select in GatewayPlanSelector | Updates local state. Does NOT auto-calculate. User must click "Guardar y calcular" to see updated margins. |
| Navigate back | Click "Volver a escenarios" link | Navigate to /escenarios. Unsaved changes: no warning (per user preference for functionality first). |

### Loading States

| State | Visual |
|-------|--------|
| Scenario list loading | Skeleton cards (3 cards with Skeleton component) |
| Scenario editor initial load | Full page skeleton (header + table rows with Skeleton) |
| Calculating margins | Spinner (Loader2) in "Guardar y calcular" button + margin columns show Skeleton |
| Saving overrides | Button shows Loader2 spinner + disabled state |

### Error States

| Error | Display |
|-------|---------|
| Failed to load scenarios | toast.error("No se pudieron cargar los escenarios") + empty state with retry |
| Failed to save overrides | toast.error("Error al guardar los overrides. Intenta de nuevo.") |
| Failed to calculate | toast.error("Error en el calculo del escenario") + margin columns show "--" |
| Scenario not found (404) | Redirect to /escenarios with toast.error("Escenario no encontrado") |
| Network error | toast.error("Error de conexion. Verifica tu internet.") |

## Financial Value Formatting

All financial values in this phase MUST use the existing formatting utilities:

| Format | Function | Example Output |
|--------|----------|----------------|
| ARS currency | `new Intl.NumberFormat('es-AR', { style: 'currency', currency: 'ARS' })` | `$ 87.000,00` |
| Selling price | `formatSellingPrice()` from products/types.ts | `$87.000` |
| Cost | `formatCost()` from products/types.ts | `$42.350,50` |
| Margin | `formatMargin()` from products/types.ts | `{ amount: "$44.649,50", percent: "105%" }` |
| Percentage | `value.toFixed(2) + '%'` | `12.50%` |

**Override price input:** Display as integer (no decimals) per CONTEXT.md decision that bulk override rounds to nearest integer. Input type="number", step="1", min="0". Displayed with `$` prefix inline (same pattern as calculadora price inputs).

**Margin difference format:** `+$X.XXX` (green, TrendingUp icon) or `-$X.XXX` (red, TrendingDown icon). Use existing DesglosePanel color conventions.
