    **Module** (`tiendanube-config.module.ts`):
    Follow CatalogsModule pattern exactly. Import TypeOrmModule.forFeature with all 5 entities. Register controller and service. Export TiendanubeConfigService (CalculadoraModule will import it in Phase 11).

    ```typescript
    @Module({
      imports: [TypeOrmModule.forFeature([TnPaymentGateway, TnGatewayRate, TnInstallmentRate, TnTaxConfig, TnPlan])],
      controllers: [TiendanubeConfigController],
      providers: [TiendanubeConfigService],
      exports: [TiendanubeConfigService],
    })
    export class TiendanubeConfigModule {}
    ```

    **Service** (`tiendanube-config.service.ts`):
    Inject all 5 repositories via @InjectRepository. Implement these methods:

    **CRITICAL — Decimal parsing contract:** All methods that return data MUST parse decimal string columns (`ratePercent`, `ivaRate`, `iibbRate`, `cptPagoNube`, `cptOtherGateways`) to `number` before returning. This happens at the service boundary so the frontend always receives numbers, never strings. Use `parseFloat()` on each decimal field. Example:
    ```typescript
    private parseRate(rate: TnGatewayRate): TnGatewayRate & { ratePercent: number } {
      return { ...rate, ratePercent: parseFloat(rate.ratePercent as string) };
    }
    ```
    Apply this pattern consistently to all entities with decimal columns.

    1. `getAll()` — Returns combined object with all config data. For gateway rates, return only the latest active rate per (gateway, paymentMethod, withdrawalDays) combination. Use DISTINCT ON or a subquery to get latest by created_at. Structure:
       ```typescript
       interface TiendanubeConfigAll {
         gateways: TnPaymentGateway[];
         rates: TnGatewayRateWithGateway[];  // latest per combo, includes gateway relation
         installments: TnInstallmentRate[];   // latest per installment count
         taxConfig: TnTaxConfig;              // latest single row
         plans: TnPlan[];
       }
       ```
       All numeric rate fields in the response are `number`, not `string`.

    2. `getGatewaysWithRates()` — Returns all active gateways with their latest rates grouped by gateway.

    3. `updateGatewayRate(gatewayId: string, dto: UpdateGatewayRateDto)` — Append-only: creates a NEW TnGatewayRate record for the given gateway + payment_method + withdrawal_days combination. Does NOT update existing records. This preserves history. Returns the new rate record.

    4. `getInstallmentRates()` — Returns latest active installment rate per installment count (1, 3, 6, 9, 12). Use DISTINCT ON (installments) ORDER BY created_at DESC.

    5. `updateInstallmentRate(installments: number, dto: UpdateInstallmentRateDto)` — Append-only: creates a new TnInstallmentRate record. Returns new record.

    6. `getTaxConfig()` — Returns the latest TnTaxConfig row (ORDER BY created_at DESC LIMIT 1).

    7. `updateTaxConfig(dto: UpdateTaxConfigDto)` — Append-only: creates a new TnTaxConfig row. Returns new row.

    8. `getPlans()` — Returns all active plans.

    9. `updatePlan(id: string, dto: UpdatePlanDto)` — Direct update on the plan row (plans are NOT append-only — there are only 4 fixed plans). Updates CPT rates.

    Important: For "latest rate" queries on append-only tables, use raw query with DISTINCT ON for efficiency:
    ```sql
    SELECT DISTINCT ON (gateway_id, payment_method, withdrawal_days) *
    FROM tn_gateway_rates
    WHERE is_active = true
    ORDER BY gateway_id, payment_method, withdrawal_days, created_at DESC
    ```
    Same pattern for installments: `DISTINCT ON (installments)`.

    **Controller** (`tiendanube-config.controller.ts`):
    Follow SuppliesController pattern for decorators (@ApiTags, @ApiBearerAuth, @Roles, @ApiOperation, @ApiResponse, ParseUUIDPipe).

    Endpoints:
    - `GET /tiendanube-config/all` — No role restriction (any authenticated user can read). Calls service.getAll().
    - `GET /tiendanube-config/gateways` — Returns gateways with latest rates.
    - `PUT /tiendanube-config/gateway-rates/:gatewayId` — @Roles(Role.ADMIN). Body: UpdateGatewayRateDto. Calls service.updateGatewayRate().
    - `GET /tiendanube-config/installments` — Returns latest installment rates.
    - `PUT /tiendanube-config/installment-rates/:installments` — @Roles(Role.ADMIN). Param is integer (installment count). Calls service.updateInstallmentRate().
    - `GET /tiendanube-config/taxes` — Returns latest tax config.
    - `PUT /tiendanube-config/taxes` — @Roles(Role.ADMIN). Calls service.updateTaxConfig().
    - `GET /tiendanube-config/plans` — Returns all active plans.
    - `PUT /tiendanube-config/plans/:id` — @Roles(Role.ADMIN). Calls service.updatePlan(). ParseUUIDPipe on id param.
