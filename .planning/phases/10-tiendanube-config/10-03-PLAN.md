<objective>
Fix the raw SQL snake_case field name bug in TiendanubeConfigService that causes NaN/null values for all rate fields, crashing the /configuracion/tiendanube page and silently breaking GatewaySelectors on the /calculadora page.

Purpose: The two raw SQL methods (getLatestGatewayRates, getInstallmentRates) return PostgreSQL column names in snake_case but all parse helpers and downstream consumers expect camelCase entity field names. This is a single root cause blocking 5 of 8 UAT tests.

Output: Fixed service with column aliases in raw SQL, cleaned up calculadora workarounds, updated test mocks.
</objective>

  <files>nemea-back/src/tiendanube-config/tiendanube-config.service.ts</files>
  <action>
    Fix the two raw SQL methods that return snake_case field names by adding explicit column aliases.

    **Method 1: `getLatestGatewayRates()` (line ~230-247)**

    Replace `gr.*` with explicit column list using camelCase aliases:

    ```sql
    SELECT DISTINCT ON (gr.gateway_id, gr.payment_method, gr.withdrawal_days)
      gr.id,
      gr.payment_method AS "paymentMethod",
      gr.withdrawal_days AS "withdrawalDays",
      gr.rate_percent AS "ratePercent",
      gr.is_active AS "isActive",
      gr.created_at AS "createdAt",
      gr.updated_at AS "updatedAt",
      gr.gateway_id AS "gatewayId",
      json_build_object(
        'id', gw.id,
        'slug', gw.slug,
        'label', gw.label,
        'isActive', gw.is_active
      ) AS gateway
    FROM tn_gateway_rates gr
    JOIN tn_payment_gateways gw ON gw.id = gr.gateway_id
    WHERE gr.is_active = true
    ORDER BY gr.gateway_id, gr.payment_method, gr.withdrawal_days, gr.created_at DESC
    ```

    IMPORTANT: Replace `row_to_json(gw.*)` with `json_build_object(...)` to produce camelCase keys in the nested gateway JSON object. `row_to_json(gw.*)` returns raw column names (is_active, created_at) which would still be snake_case. `json_build_object` lets us control the key names explicitly.

    Also update the `getGatewaysWithRates()` method filter (line ~127-131). Remove the `(r as { gateway_id?: string }).gateway_id` fallback since after the fix `r.gateway.id` will always work. The filter should just be:
    ```typescript
    rates: rates.filter((r) => r.gateway?.id === gateway.id),
    ```

    **Method 2: `getInstallmentRates()` (line ~158-167)**

    Replace `SELECT DISTINCT ON (installments) *` with explicit columns:

    ```sql
    SELECT DISTINCT ON (installments)
      id,
      installments,
      rate_percent AS "ratePercent",
      is_active AS "isActive",
      created_at AS "createdAt",
      updated_at AS "updatedAt"
    FROM tn_installment_rates
    WHERE is_active = true
    ORDER BY installments, created_at DESC
    ```

    **What NOT to change:**
    - Do NOT modify the 4 parse helpers (parseGatewayRate, parseInstallmentRate, parseTaxConfig, parsePlan). They already read camelCase fields. The fix is making the SQL produce camelCase keys so the helpers work as-is.

    - Do NOT modify the update methods — they use TypeORM .save() which returns entities with camelCase.
    - Do NOT modify DTOs, controller, or entities.
