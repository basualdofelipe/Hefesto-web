    **ModeToggle.tsx** — Forward/Inverse toggle using Shadcn Tabs or custom toggle (Shadcn ToggleGroup). Two options: "Precio -> Ganancia" and "Ganancia -> Precio". Controlled via `mode` / `onModeChange` props.

    **ProductSelector.tsx** — Product combobox/select. Receives products array (fetched server-side). Groups products by type.name for readability (per CONTEXT.md "grouped by type > name"). When a product is selected, calls `onProductSelect(product)` — the parent uses product.id to send to backend (which fetches cost from DB per CALC-03). Also displays the product cost inline after selection: "Costo: $6,534.48" (from product.cost). If product.cost is null, show warning "Sin costo definido". Use Shadcn Select with groups, or Shadcn Command (combobox) if the product list may be long.

    **GatewaySelectors.tsx** — Cascading selectors. Receives TiendanubeConfigAll as prop. State management:
    1. Gateway selector (Pago Nube / MP / MODO — only active gateways)
    2. When gateway changes, filter `rates` by selected gateway to get unique paymentMethod values -> populate payment method selector. Use PAYMENT_METHOD_LABELS from tiendanube-config/types.ts for display.
    3. When paymentMethod changes, filter rates by gateway + method to get unique withdrawalDays values -> populate withdrawal selector (show as "X dias").
    4. Cuotas selector (1, 3, 6, 9, 12) — from config.installments.
    5. Plan selector: default = 'esencial' (per Phase 10 decision). A toggle/switch "Simular otro plan" reveals the plan selector. When hidden, use the default plan from config (per CONTEXT.md "inherit from config + override toggle").

    All selectors use Shadcn Select. Expose all selected values via `onConfigChange({ gatewaySlug, paymentMethod, withdrawalDays, installments, planSlug })` callback.

    **DesglosePanel.tsx** — Right column. Receives CalcResult (or CalcInverseResult) + mode + costoProducto as props. Renders all intermediate values in a clean list:
    - In inverse mode: show "Precio de venta necesario: $X" at top (highlighted blue, matching prototype style)
    - Total pagado por cliente
    - Comision pasarela (tasaBase% + IVA = tasaConIVA%) — negative, red
    - Financiacion cuotas (only if installments > 1) — negative, red
    - CPT Tiendanube — negative, red (new vs prototype)
    - Retenciones IIBB — negative, red
    - Neto recibido
    - Separator
    - IVA debito fiscal — negative
    - IVA credito producto — positive (sub-item)
    - IVA credito comision — positive (sub-item)
    - IVA neto a pagar — negative
    - Costo producto + IVA — negative
    - GANANCIA REAL (margen X%) — highlighted green, large font
    Use ARS formatting: `new Intl.NumberFormat('es-AR', { style: 'currency', currency: 'ARS' })`. Per prototype formatting.
    Show "Valores aproximados..." disclaimer at bottom.

    **CalculadoraClient.tsx** — Main client component. State:
    - mode: 'forward' | 'inverse'
    - selectedProduct: Product | null
    - precioVenta: string (input state)
    - costoEnvio: string (input state)
    - gananciaDeseada: string (input state)
    - result: CalcResult | CalcInverseResult | null
    - error: string | null
    - loading: boolean
    - gatewayConfig: { gatewaySlug, paymentMethod, withdrawalDays, installments, planSlug }

    Calculation trigger: debounce 300ms after any input change. When all required fields are filled (product selected, price/profit entered, gateway configured), call the backend:
    - Forward mode: POST /api/calculadora/forward with { productId, precioVenta: parseFloat(precioVenta), costoEnvio: parseFloat(costoEnvio) || 0, ...gatewayConfig }
    - Inverse mode: POST /api/calculadora/inverse with { productId, gananciaDeseada: parseFloat(gananciaDeseada), costoEnvio: parseFloat(costoEnvio) || 0, ...gatewayConfig }
    - On 400 response (inverse edge cases), extract message and set error state. Display error as Shadcn Alert (variant="destructive").

    Use `useSession()` to get accessToken for apiClientFetch. Use `useCallback` + `useRef` for debounce (not setTimeout directly — cleanup on unmount).

    Two-column layout (per CONTEXT.md): On lg+ screens, `grid grid-cols-2 gap-6`. On mobile, single column (inputs then results stacked). Left column: ModeToggle, ProductSelector, price/profit input (Shadcn Input with $ prefix), envio input, GatewaySelectors. Right column: DesglosePanel (or placeholder "Selecciona un producto y completa los datos" when no result).

    Wrap input sections in Shadcn Card components for visual grouping (matching the app's existing style).
