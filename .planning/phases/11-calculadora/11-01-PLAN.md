    **CalculadoraService** — stateless, injectable:
    - Constructor injects TiendanubeConfigService + CostsService
    - `resolveRates(config, gatewaySlug, paymentMethod, withdrawalDays, installments, planSlug?)` — private helper. From TiendanubeConfigAll, finds the matching gateway rate (ratePercent), installment rate, tax config (ivaRate, iibbRate as fractions not percentages), and CPT (from plan + gateway). Returns a flat object with all needed rates as numbers. Throws NotFoundException if gateway/rate combo not found.
    - `calcForward(params)` — PURE MATH, adapted from prototype. The 14-step formula from `_archive/calculadora/calculadora-tiendanube.jsx` lines 156-211. Key adaptation: the prototype uses `planData.tarjeta[plazoRetiro]` — the new version uses `resolveRates()` to get `gatewayRate` from DB. Add CPT step (line between IIBB and netoRecibido): `cpt = totalCliente * (cptRate / 100)`. Update netoRecibido to subtract CPT. Round only gananciaReal and margen at the end: `Math.round(value * 100) / 100`. Per CONTEXT.md decision and research PITFALL-1: NO intermediate rounding.
    - `calcInverse(params)` — Binary search over calcForward. Epsilon convergence: `while (high - low > 0.01 && iterations < 100)`. Validate edge cases BEFORE search: costoProducto <= 0 returns CalcError, gananciaDeseada < 0 returns CalcError. Upper bound check: if calcForward(high).gananciaReal < gananciaDeseada, return CalcError "Ganancia inalcanzable". Upper bound = costoProducto * 20 (same as prototype, but if costoProducto is very small, use max(costoProducto * 20, 100000)).
    - `calcBatch(dto)` — Loads TN config once (1 query via getAll), loads all product costs (2 queries via calculateAll), loads all products with prices (2 queries via findAll). Runs calcForward in-memory loop for each product that has a currentPrice. Returns CalcBatchItem[]. Total: ~5 queries + O(n) in-memory math. Per CALC-04: no more than 6 SQL queries.

    For calcBatch, inject ProductsService (or use products repo) to get product names and current prices. Import ProductsModule. The service needs: product id, display name (type.name + ' ' + name.name + ' ' + finish.name), cost, currentPrice.
