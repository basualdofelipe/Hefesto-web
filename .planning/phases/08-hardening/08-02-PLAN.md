**Step 2: Create shared formatters `nemea-front/src/lib/formatters.ts`**:

```typescript
export function formatDate(iso: string | null): string {
  if (!iso) return '-';
  return new Date(iso).toLocaleDateString('es-AR', {
    day: '2-digit',
    month: '2-digit',
    year: 'numeric',
  });
}
```

CRITICAL: The date format options `{ day: '2-digit', month: '2-digit', year: 'numeric' }` are REQUIRED. Without them, dates display as "26/3/2026" instead of "26/03/2026" — a visual regression from the current behavior. All 4 existing formatDate implementations include these options.

The formatDate accepts `string | null` (the superset signature — some callsites pass null, others don't).

Do NOT create a shared `formatAmount` function. The `expenses/types.ts` already has a `formatAmount` with different behavior (returns "1.234,56" without `$` prefix). Creating a shared version with `$` prefix would be a naming collision footgun. Leave the existing `formatAmount` in `expenses/types.ts` untouched.
