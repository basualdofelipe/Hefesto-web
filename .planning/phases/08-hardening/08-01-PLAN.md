1. In `nemea-back/src/users/users.controller.ts`, add a new PATCH endpoint for toggling user status:
   - Import `Patch` and `NotFoundException` from `@nestjs/common`
   - Add a `PATCH :id/toggle-status` route with `@Roles(Role.ADMIN)` decorator
   - Implementation: call `usersService.findById(id)`, throw NotFoundException if not found, then call `usersService.deactivate(id)` if user.isActive is true, else call `usersService.activate(id)`. Return the updated user by calling `usersService.findById(id)` again.
   - Add Swagger decorators: `@ApiOperation({ summary: 'Alternar estado activo/inactivo del usuario' })`, `@ApiResponse` for 200, 404, 403
   - IMPORTANT: Do NOT add `@Roles(Role.ADMIN)` to the class level -- it must stay on individual mutation methods, because GET endpoints on other controllers are intended to be read-accessible to all authenticated users (USER role investors need to read data)

2. In `nemea-back/src/supplies/supplies.service.ts`, delete the `deactivateBySupplier` method (lines 275-282). This method is never called by any controller or service -- confirmed by grep. It is dead code.
