1. **Update `nemea-back/src/users/users.controller.ts`**:
   - Add `Patch` and `NotFoundException` to the NestJS imports
   - Replace the existing `@Delete(':id') remove()` method with a `@Patch(':id/toggle-status')` endpoint:
     ```typescript
     @Patch(':id/toggle-status')
     @Roles(Role.ADMIN)
     @ApiOperation({ summary: 'Toggle user active status (ADMIN only)' })
     @ApiResponse({ status: 200, description: 'User status toggled' })
     @ApiResponse({ status: 403, description: 'Forbidden — requires ADMIN role' })
     async toggleStatus(@Param('id', ParseUUIDPipe) id: string): Promise<User> {
       const user = await this.usersService.findById(id);
       if (!user) {
         throw new NotFoundException('Usuario no encontrado');
       }
       if (user.isActive) {
         await this.usersService.deactivate(id);
       } else {
         await this.usersService.activate(id);
       }
       const updated = await this.usersService.findById(id);
       if (!updated) {
         throw new NotFoundException('Usuario no encontrado');
       }
       return updated;
     }
     ```
   - IMPORTANT: Do NOT use `as Promise<User>` cast on the second findById call. Use a proper null check with NotFoundException as shown above. This handles the (extremely unlikely but type-safe) case where the user is deleted between the toggle and the re-fetch.
   - KEEP the existing `@Post()` create endpoint and `@Get()` findAll endpoint unchanged
   - REMOVE the `@Delete(':id') remove()` endpoint — we use soft delete (toggle-status), not hard delete
   - Remove `Delete` from the NestJS imports since it's no longer used
