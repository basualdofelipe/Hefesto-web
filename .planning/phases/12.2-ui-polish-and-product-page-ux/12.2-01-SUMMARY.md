# Phase 12.2 Plan 01: Sort Order for Product Sizes Summary

**SMALLINT sort_order column added to product_sizes with seeded logical ordering (Unico/XS/S/Chico/M/Mediano/L/Grande!), TypeORM entity updated, and CatalogsService returns sizes in display order**

## Accomplishments
- Migration `1773100000000-AddSortOrderToProductSizes` adds `sort_order SMALLINT NOT NULL DEFAULT 0` column and seeds all 8 existing sizes with logical display order
- `ProductSize` entity gains `sortOrder` TypeORM column mapping — new sizes created via catalog CRUD default to sort_order = 0
- `CatalogsService.findAll` returns `product-sizes` ordered by `sortOrder ASC, name ASC` (secondary sort handles ties at sort_order = 0)
- Numeric belt sizes handled via `CAST(name AS INTEGER) WHERE name ~ '^[0-9]+$'` pattern for future-proof natural ordering

## Files Created/Modified
- `nemea-back/src/database/migrations/1773100000000-AddSortOrderToProductSizes.ts` - Migration: adds sort_order column, seeds 8 named sizes with logical display order, handles numeric belt sizes via regex
- `nemea-back/src/catalogs/entities/product-size.entity.ts` - Added `sortOrder` column decorator (name: 'sort_order', smallint, default: 0)
- `nemea-back/src/catalogs/catalogs.service.ts` - findAll special-cases 'product-sizes' dimension to use `productSizeRepo` with `{ sortOrder: 'ASC', name: 'ASC' }` ordering

## Decisions Made
- Used `this.productSizeRepo.find(...)` directly instead of the generic `repo.find({ ... } as any)` to avoid the `any` cast that CLAUDE.md and ESLint forbid. The typed `productSizeRepo: Repository<ProductSize>` has the `sortOrder` field and allows the order clause without casting.
- Sort_order spacing uses 5 between existing named sizes (0, 5, 10, 15, 20, 25, 30, 35) rather than uniform 10 — this gives more room for future insertions. 'Unico' stays at 0 (matches its SKU meaning of a single-size product).
