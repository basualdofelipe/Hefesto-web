<decisions>
## Implementation Decisions

### Cascada de Productos
- **D-01:** ProductNameGroup and ProductFinishGroup default to collapsed (`useState(false)`). ProductTypeGroup stays open. User sees types with product counts, drills down on demand.
- **D-02:** No structural changes to the 4-level hierarchy (Type > Name > Finish > Row). Only the default open/closed state changes.

### Home Page
- **D-03:** Replace the brand splash with quick-link cards (Productos, Calculadora, Insumos, Gastos). Cards are permission-aware using `usePermissions()` — only show what the user's role allows.
- **D-04:** RSC component, no API calls. Cards link to the same routes as sidebar items.

### Logo and Branding
- **D-05:** Logo assets are in `nemea-front/public/brand/`. Three files: logo (NEMEA text), isotipo (lion), leon+color (lion with burgundy circle background).
- **D-06:** Sidebar expanded state: show logo PNG (NEMEA text). Sidebar collapsed state: show isotipo PNG (lion).
- **D-07:** Login page: show isotipo (lion) or leon+color replacing the current text branding.

### Typography
- **D-08:** Mechanical audit of all `font-medium` (60 uses) and `font-bold` (3 uses). Rule: headers/labels/badges/table-headers → `font-semibold`, body/descriptions/placeholders → `font-normal`. Result: only weights 400 and 600.

### Size Ordering
- **D-09:** Add `sort_order` column to sizes catalog table via migration. Sizes display in sort_order instead of alphabetically. Numeric sizes (cinturones: 75, 80, 85...) get sequential sort_order values.

### Skeletons
- **D-10:** No skeleton changes in this phase. Current text placeholders ("Cargando...", "—") are acceptable for now.

### Table Column Alignment (folded todo)
- **D-11:** Fix column alignment and visual polish across tables. Claude's discretion on specific fixes.
