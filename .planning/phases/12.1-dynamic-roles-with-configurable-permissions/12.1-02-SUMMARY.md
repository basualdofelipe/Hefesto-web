## Decisions Made

1. **TiendanubeConfig GETs open to any authenticated user** — GET `all`, `gateways`, `installments`, `taxes`, `plans` have no @RequirePermission. A user with `can_use_calculator` but not `can_view_products` needs TN config data. PermissionsGuard returns true when no permissions metadata is set.
2. **Admin bypass via canManageUsers** — `ScenariosService.remove()` now accepts `Permissions` and checks `permissions.canManageUsers` instead of `role === Role.ADMIN`. Semantic alignment: user management = admin-level access.
3. **Defense in depth on GETs** — Every GET endpoint in all 9 controllers (except TN config) requires the appropriate view permission. Prevents unauthorized data access even with valid JWTs.
4. **Unused Permissions import removed from scenarios.controller** — `JwtUser` already types `permissions: Permissions`; no explicit import of `Permissions` needed in the controller file.
