### WR-02: Migration down() will fail for custom roles with a PostgreSQL cast error

**Files modified:** `nemea-back/src/database/migrations/1773000000000-CreateRolesAndMigrateUsers.ts`
**Commit:** `5f25d29`
**Applied fix:** Added a safety `UPDATE` step between the `LOWER("role")` step and the `ALTER COLUMN ... USING` cast in `down()`. The new query remaps any role value not in `('admin', 'user')` to `'user'` before the enum cast, preventing a PostgreSQL "invalid input value for enum" crash when custom roles exist in production.

---

### WR-03: No backend protection against self-deactivation

**Files modified:** `nemea-back/src/users/users.controller.ts`
**Commit:** `d6d8e02`
**Applied fix:** Added `BadRequestException` to NestJS imports. Imported `CurrentUser` and `JwtUser` from the auth decorator. Added `@CurrentUser() currentUser: JwtUser` as a second parameter to `toggleStatus()`. Added an early guard that throws `BadRequestException('No puedes desactivar tu propia cuenta')` when `id === currentUser.id`.
