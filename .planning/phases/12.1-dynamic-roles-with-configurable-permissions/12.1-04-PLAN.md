**1. Create page.tsx** (`nemea-front/src/app/(app)/roles/page.tsx`):
Server component. Pattern matches existing pages (catalogos, usuarios):
```typescript
import type { ReactElement } from 'react';
import { auth } from '@/auth';
import { redirect } from 'next/navigation';
import { apiFetch } from '@/lib/api';
import { RolesClient } from './RolesClient';

// Role type for API response
interface RoleRow {
  id: string;
  name: string;
  description: string | null;
  isSystem: boolean;
  canViewProducts: boolean;
  canEditProducts: boolean;
  canViewSupplies: boolean;
  canEditSupplies: boolean;
  canViewExpenses: boolean;
  canEditExpenses: boolean;
  canUseCalculator: boolean;
  canManageScenarios: boolean;
  canViewDashboard: boolean;
  canManageConfig: boolean;
  canManageUsers: boolean;
  userCount: number;
}

export default async function RolesPage(): Promise<ReactElement> {
  const session = await auth();
  if (!session?.user?.permissions?.canManageUsers) {
    redirect('/');
  }
  const rolesRes = await apiFetch<{ data: RoleRow[] }>('/api/roles');
  return (
    <div className='space-y-6'>
      <div className='flex items-center justify-between'>
        <div>
          <h1 className='text-2xl font-semibold tracking-tight'>Roles</h1>
          <p className='text-muted-foreground text-sm'>
            Gestiona los roles y permisos del sistema.
          </p>
        </div>
      </div>
      <RolesClient initialRoles={rolesRes.data} />
    </div>
  );
}
```

**2. Create RolesClient.tsx** (`nemea-front/src/app/(app)/roles/RolesClient.tsx`):
Client component with 'use client' directive. Follows the catalog/users page pattern but with the UI-SPEC specifics.

**State:**
- `roles: RoleRow[]` — initialized from props
- `dialogOpen: boolean` — for create/edit dialog
- `editingRole: RoleRow | null` — null = create mode, set = edit mode
- `deleteDialogOpen: boolean` — for delete confirmation
- `deletingRole: RoleRow | null` — role being deleted
- `isSubmitting: boolean` — loading state for form submit
- `isDeleting: boolean` — loading state for delete

**Form (zod + react-hook-form):**
```typescript
const roleSchema = z.object({
  name: z.string().min(1, 'El nombre es obligatorio').max(100, 'Maximo 100 caracteres'),
  description: z.string().max(255, 'Maximo 255 caracteres').optional().or(z.literal('')),
  canViewProducts: z.boolean().default(false),
  canEditProducts: z.boolean().default(false),
  canViewSupplies: z.boolean().default(false),
  canEditSupplies: z.boolean().default(false),
  canViewExpenses: z.boolean().default(false),
  canEditExpenses: z.boolean().default(false),
  canUseCalculator: z.boolean().default(false),
  canManageScenarios: z.boolean().default(false),
  canViewDashboard: z.boolean().default(false),
  canManageConfig: z.boolean().default(false),
  canManageUsers: z.boolean().default(false),
});
```

**Table (per UI-SPEC):**
5 columns: Nombre, Descripcion, Usuarios, Permisos, Acciones.
- Nombre: role name in `font-medium`. If isSystem, show Badge variant="outline" with text "Sistema" next to name.
- Descripcion: description text or `"-"` in `text-muted-foreground`
- Usuarios: numeric user count, center-aligned
- Permisos: Badge showing "{N}/11" where N = count of true permission flags.
  - N=11: Badge variant="default" (primary bg)
  - N=1-10: Badge variant="secondary"
  - N=0: Badge variant="outline"
- Acciones: "Editar" Button variant="outline" size="sm" + "Eliminar" Button variant="outline" size="sm" className="text-destructive" (only for non-system roles)

**Create/Edit Dialog (per UI-SPEC):**
Dialog with DialogHeader, form content, DialogFooter.
- Title: "Crear rol" (create) / "Editar rol: {name}" (edit)
- Form sections separated by Separator:
  1. Info: name Input + description Input
  2. "Productos": canViewProducts Switch + canEditProducts Switch
  3. "Insumos y proveedores": canViewSupplies + canEditSupplies
  4. "Gastos": canViewExpenses + canEditExpenses
  5. "Herramientas": canUseCalculator + canManageScenarios + canViewDashboard
  6. "Administracion": canManageConfig + canManageUsers

Each permission row: `flex items-center justify-between py-2` with Label and Switch.
Section headers: `text-sm font-semibold text-muted-foreground uppercase tracking-wide pt-2 pb-1`
Form area scrollable: `max-h-[70vh] overflow-y-auto`

On submit (create):
- POST /api/roles with form data via `apiClientFetch`
- On success: toast.success('Rol creado'), close dialog, refresh roles from server
- On error: toast.error(error message or 'Error al crear rol')

On submit (edit):
- PATCH /api/roles/:id with form data
- On success: toast.success('Rol actualizado'), close dialog, refresh
- On error: toast.error('Error al guardar cambios')

Use `apiClientFetch` (client-side fetch with auth) for mutations. For refresh, call `router.refresh()` to re-run server component.

**Delete Confirmation (AlertDialog per UI-SPEC):**
- AlertDialogTitle: "Eliminar rol"
- AlertDialogDescription: `Se eliminara el rol "{roleName}". Los usuarios asignados a este rol quedaran sin acceso. Esta accion no se puede deshacer.`
- Cancel: "Cancelar"

- On confirm: DELETE /api/roles/:id, toast.success('Rol eliminado'), refresh

**Loading states per UI-SPEC:**

- Delete: Loader2 spinner + disabled on confirm button
- Table loading: not needed (data comes from server component props)

**Empty state:** If no custom roles (only system roles), show centered message: "No hay roles personalizados. Crea uno para asignar permisos especificos."
