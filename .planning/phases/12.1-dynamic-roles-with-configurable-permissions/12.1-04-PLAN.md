**REVIEW FIX (Gemini concern: permission dependency in UI):** Implement auto-check/uncheck logic for view/edit permission pairs. When a `can_edit_X` switch is toggled ON, automatically check the corresponding `can_view_X`. When a `can_view_X` switch is toggled OFF, automatically uncheck the corresponding `can_edit_X`.

The 3 view/edit pairs:
- canViewProducts <-> canEditProducts
- canViewSupplies <-> canEditSupplies
- canViewExpenses <-> canEditExpenses

Implementation: Use `watch` from react-hook-form to observe changes, and `setValue` to enforce the dependency:
```typescript
function handlePermissionToggle(field: keyof typeof roleSchema.shape, value: boolean): void {
  setValue(field, value);

  const viewEditPairs: [keyof Permissions, keyof Permissions][] = [
    ['canViewProducts', 'canEditProducts'],
    ['canViewSupplies', 'canEditSupplies'],
    ['canViewExpenses', 'canEditExpenses'],
  ];

  for (const [viewKey, editKey] of viewEditPairs) {
    if (field === editKey && value === true) {
      // Turning ON edit -> auto-check view
      setValue(viewKey, true);
    }
    if (field === viewKey && value === false) {
      // Turning OFF view -> auto-uncheck edit
      setValue(editKey, false);
    }
  }
}
```

Each Switch `onCheckedChange` calls `handlePermissionToggle(fieldName, newValue)` instead of directly calling `setValue`.
