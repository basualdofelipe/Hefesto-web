**1. Fix ScenariosService admin bypass** (CRITICAL — Pitfall 6 from research):
The `remove()` method currently does:
```typescript
async remove(id: string, userId: string, role: string): Promise<void> {
  const where =
    role === Role.ADMIN ? { id } : { id, user: { id: userId } };
```

Change the method signature to accept `permissions: Permissions` instead of `role: string`:
```typescript
import type { Permissions } from '../../common/types/permission';

async remove(id: string, userId: string, permissions: Permissions): Promise<void> {
  const where = permissions.canManageUsers ? { id } : { id, user: { id: userId } };
```

Remove the import of `Role` from `../../common/types/role.enum`.

The admin bypass now checks `canManageUsers` permission — if the user can manage users, they can delete any scenario (admin-level access semantic).
