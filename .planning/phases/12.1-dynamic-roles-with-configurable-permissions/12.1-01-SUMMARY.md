# Phase 12.1 Plan 01: Backend Permission System Summary

Backend permission system with Role entity (11 boolean flags), breaking migration, RolesModule CRUD with admin lockout prevention, PermissionsGuard replacing RolesGuard, and JWT with embedded permissions via extractPermissions helper.

## What Was Built

### Task 1: Role entity, permission constants, RolesModule CRUD, and breaking migration
- **Permission type system** (`permission.ts`): `Permission` literal union type (11 values), `Permissions` interface (camelCase booleans), `NO_PERMISSIONS` constant, `PERMISSION_TO_CAMEL` map, `extractPermissions` null-safe helper with explicit object type guard
- **Role entity**: 11 boolean permission columns + isSystem + name + description + userCount virtual property, extends BaseEntity (UUID + timestamps)
- **DTOs**: CreateRoleDto with @Transform defaults, UpdateRoleDto via PartialType
- **RolesService**: Full CRUD with system role protection, admin lockout prevention (cannot remove canManageUsers from ADMIN), user count via loadRelationCountAndMap
- **RolesController**: 4 endpoints (GET, POST, PATCH, DELETE) initially protected with @Roles(ADMIN), later updated to @RequirePermission
- **Breaking migration**: 12-step migration (create roles table, seed ADMIN/USER, add role_id column, create index, migrate users with parameterized queries, set NOT NULL, add FK, drop old column and enum type)
- **AppModule**: RolesModule registered after UsersModule

### Task 2: PermissionsGuard, @RequirePermission decorator, auth system refactor
- **@RequirePermission decorator**: Type-safe with `Permission[]` parameter (compile-time typo detection)
- **PermissionsGuard**: Checks `PERMISSIONS_KEY` metadata, converts snake_case permissions to camelCase via `PERMISSION_TO_CAMEL`, throws ForbiddenException
- **JwtUser interface**: Now has `permissions: Permissions` instead of `role: string`
- **JwtPayload**: Includes `permissions: Permissions` for frontend consumption
- **JwtStrategy.validate()**: Uses `extractPermissions(user.role)` to read fresh permissions from DB on every request
- **AuthService**: JWT signed with permissions, response returns permissions object
- **AuthResponseDto**: `permissions` field replaces `role` string
- **User entity**: `role` property is now `ManyToOne(() => Role, { eager: true })` with `@JoinColumn({ name: 'role_id' })`
- **UsersModule**: `TypeOrmModule.forFeature([User, Role])` for direct Role repo access
- **UsersService.create()**: Validates role existence via `roleRepository.findOne` before FK assignment
- **CreateUserDto**: `roleId?: string` with `@IsUUID()` replaces `role?: Role` enum
- **AuthModule**: `PermissionsGuard` replaces `RolesGuard` as `APP_GUARD`
- **RolesController**: Updated to use `@RequirePermission('can_manage_users')` on all endpoints
- **RolesGuard**: Deprecated but kept for compilation (Plan 02 will remove)
- **All spec files**: Updated mock objects to use Role entity instead of Role enum
