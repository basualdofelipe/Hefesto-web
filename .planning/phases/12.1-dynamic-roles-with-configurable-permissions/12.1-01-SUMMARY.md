## Decisions Made

1. **Permission flags as booleans on Role entity** — simpler and faster than a separate permissions table. 11 flags cover all app modules.
2. **Backend re-reads permissions from DB on every request** — JwtStrategy.validate() calls findActiveByEmail() with eager role load. JWT carries permissions only for frontend consumption.
3. **UsersModule imports Role entity directly** via `TypeOrmModule.forFeature([User, Role])` — avoids circular dependency between UsersModule and RolesModule.
4. **RolesGuard retained with backward-compat logic** — maps `canManageUsers → ADMIN`, any user → `USER`. Deleted in Plan 02 after all controllers migrate to `@RequirePermission`.
5. **ScenariosController.remove** uses `user.permissions.canManageUsers` instead of role string comparison to determine if user can delete any scenario.
