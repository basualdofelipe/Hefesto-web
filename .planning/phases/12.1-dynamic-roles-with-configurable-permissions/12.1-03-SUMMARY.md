## Decisions Made

1. **Stale JWT detection via key deletion** — Checks `token.role !== undefined && token.permissions === undefined` to identify pre-migration tokens. Deletes `token.role` key and sets `token.backendToken = undefined`, causing the user to re-authenticate on next navigation. Simpler than rotating NEXTAUTH_SECRET (which would invalidate all valid sessions).

2. **AppSidebar conditional group rendering** — Each nav group is conditionally rendered based on specific permission flags rather than showing/hiding individual items. Admin group requires `canManageUsers || canManageConfig`; Datos base requires `canViewProducts || canViewSupplies`; etc.

3. **UsersClient.tsx interim UserRow.role union type** — `role: { id: string; name: string } | string` with `typeof user.role === 'object' ? user.role.name : user.role` display. Plan 04 replaces this with the full RoleOption type and dynamic role Select from API.

4. **canEdit prop as primary, usePermissions() as fallback** — ProductTable and SupplyTable use `canEditProp ?? canEditProducts` (via `??` operator not `||`), ensuring server-side prop takes precedence and the hook is only used when prop is explicitly `false`/undefined.
