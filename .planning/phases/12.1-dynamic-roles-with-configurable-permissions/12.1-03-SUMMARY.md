# Phase 12.1 Plan 03: Frontend Permission System Summary

Full frontend permission migration: Permissions type with 11 flags, usePermissions() hook, NextAuth session extension with stale JWT detection, middleware ROUTE_PERMISSIONS map with dev logging, and all 26 frontend files migrated from isAdmin to granular canEdit/canView permission props.

## What Was Built

### Task 1: Permissions type, usePermissions() hook, NextAuth/auth.ts/middleware updates
- **Permissions type** (`permissions.ts`): `Permissions` interface (11 booleans), `NO_PERMISSIONS` constant (all false), `PERMISSION_KEYS` const array with `satisfies` constraint
- **usePermissions() hook**: Client hook returning `session.user.permissions ?? NO_PERMISSIONS`
- **NextAuth types**: `permissions: Permissions` replaces `role: string` in Session.user, JWT, and User interfaces
- **auth.ts callbacks**: JWT callback forwards `permissions` from backend response; stale JWT detection clears old `role` key + `backendToken` to force re-auth; session callback maps `token.permissions` to `session.user.permissions` with `NO_PERMISSIONS` fallback
- **middleware.ts**: `ROUTE_PERMISSIONS` map (10 routes) with `getRequiredPermission` helper for prefix matching; development-only warning when authenticated user has undefined permissions

### Task 2: Migrate all 16 files + sidebar + interim fix + delete useIsAdmin
- **8 server pages**: Replaced `session.user.role === 'admin'` with specific permission checks (`canEditProducts`, `canEditSupplies`, `canEditExpenses`, `canManageUsers`, `canManageConfig`)
- **14 client components**: Renamed `isAdmin: boolean` prop to `canEdit: boolean` across products (6 files), supplies (3 files), suppliers (1 file), expenses (2 files), catalogs (2 files)
- **ProductTable/SupplyTable**: Special fallback pattern -- `canEdit ?? canEditProducts` (or `canEditSupplies`) via `usePermissions()` hook
- **ScenarioListClient**: Replaced `useIsAdmin()` with `usePermissions()`, uses `canManageUsers` for admin delete capability
- **AppSidebar**: Full rewrite with `usePermissions()` -- Finanzas visible when `canViewExpenses`, Herramientas when `canUseCalculator || canManageScenarios`, Productos when `canViewProducts`, Datos base when `canViewProducts || canViewSupplies`, Admin when `canManageUsers || canManageConfig`. New Roles nav item with Shield icon at `/roles`
- **/usuarios interim fix**: `UserRow.role` typed as `string | { id: string; name: string }` union, display uses `typeof user.role === 'object' ? user.role.name : user.role` for both old and new shapes. Create user dialog role Select left untouched (Plan 04 scope)
- **useIsAdmin.ts deleted**: Zero remaining imports confirmed before deletion
