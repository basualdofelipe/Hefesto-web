## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` (Tailwind default) |
| Page heading | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold tracking-tight` |
| Page subtitle | 14px | 400 (regular) | 1.5 | `text-muted-foreground text-sm` |

Two weights declared: 400 (regular) for all body, label, and subtitle text; 600 (semibold) for headings and section headers. Note: `font-medium` (500) may appear via Shadcn component internal defaults (e.g., Label, TableHead) but is not a named typographic role in this contract.

Source: All 10 existing pages use `text-2xl font-semibold tracking-tight` for h1 and `text-muted-foreground text-sm` for subtitle. No deviation.

## Page Layout: /roles

**Focal point:** The roles table is the primary focal element; it occupies the majority of the viewport and the "Crear rol" CTA in the page header is the primary action entry point.

### Page Structure

```
[Page Header]
  h1: "Roles"
  subtitle: "Gestiona los roles y permisos del sistema."
  [Right-aligned] Button: "Crear rol"

[Roles Table]
  Columns: Nombre | Descripcion | Usuarios | Permisos | Acciones
  Row pattern:
    - Nombre: role name text
    - Descripcion: description or "-" in muted-foreground
    - Usuarios: numeric count
    - Permisos: Badge with "{N}/11" + color coding
    - Acciones: "Editar" button (outline) + "Eliminar" button (destructive outline, hidden for system roles)

[Empty State]
  (centered in table body, colSpan full)
  "No hay roles personalizados. Crea uno para asignar permisos especificos."
```

### Roles Table Spec

| Column | Width | Content | Alignment |
|--------|-------|---------|-----------|
| Nombre | auto | Role name, `font-medium` | left |
| Descripcion | auto (flex grow) | Description text or `"-"` in `text-muted-foreground` | left |
| Usuarios | 100px | Numeric count | center |
| Permisos | 120px | Badge: `"{N}/11"` | center |
| Acciones | 200px | Edit + Delete buttons | right |

### Permission Count Badge Colors

| Count | Badge Variant | Meaning |
|-------|---------------|---------|
| 11/11 | `default` (primary bg) | Full access |
| 1-10/11 | `secondary` | Partial access |
| 0/11 | `outline` | No access |

### System Role Indicators

- ADMIN and USER roles display a small `Badge variant="outline"` with text "Sistema" next to the role name
- System roles show "Editar" button (permissions are editable) but NO "Eliminar" button
- Custom roles show both "Editar" and "Eliminar" buttons

## Dialog: Create/Edit Role

### Layout

```
[DialogHeader]
  Title: "Crear rol" / "Editar rol: {name}"

[Form]
  [Section: Info]
    Input: Nombre del rol (required)
    Input: Descripcion (optional)

  [Separator]

  [Section: Permisos de productos]
    Switch + Label: "Ver productos" (canViewProducts)
    Switch + Label: "Editar productos" (canEditProducts)

  [Separator]

  [Section: Permisos de insumos]
    Switch + Label: "Ver insumos y proveedores" (canViewSupplies)
    Switch + Label: "Editar insumos y proveedores" (canEditSupplies)

  [Separator]

  [Section: Permisos de gastos]
    Switch + Label: "Ver gastos" (canViewExpenses)
    Switch + Label: "Editar gastos" (canEditExpenses)

  [Separator]

  [Section: Herramientas]
    Switch + Label: "Usar calculadora" (canUseCalculator)
    Switch + Label: "Gestionar escenarios" (canManageScenarios)
    Switch + Label: "Ver dashboard" (canViewDashboard)

  [Separator]

  [Section: Administracion]
    Switch + Label: "Configurar Tiendanube" (canManageConfig)
    Switch + Label: "Gestionar usuarios y roles" (canManageUsers)

[DialogFooter]
  Button (outline): "Cancelar"
  Button (primary): "Crear rol" / "Guardar cambios"
```

### Permission Toggle Spec

Each permission row uses this layout:

```
[flex items-center justify-between py-2]
  [Label] text-sm font-normal — permission display name
  [Switch] — bound to the boolean flag
```

- Section headers use `text-sm font-semibold text-muted-foreground uppercase tracking-wide` with `pt-2 pb-1`
- The dialog body should scroll if content overflows: `max-h-[70vh] overflow-y-auto` on the form area
- Permission groups separated by `Separator` component

### Form Validation (zod)

| Field | Rule | Error Message |
|-------|------|---------------|
| name | `z.string().min(1).max(100)` | "El nombre es obligatorio" / "Maximo 100 caracteres" |
| description | `z.string().max(255).optional()` | "Maximo 255 caracteres" |
| (11 boolean flags) | `z.boolean().default(false)` | (no validation needed, defaults to false) |

## Interaction States

### Roles Table

| State | Visual Treatment |
|-------|------------------|
| Loading | Skeleton rows (3 rows, matching column widths) |
| Empty | Centered message in table body, `text-muted-foreground h-16 text-center` |
| Hover row | Default Shadcn TableRow hover (muted background) |
| System role | "Sistema" badge next to name, no delete button |
| Custom role | Edit + Delete buttons visible |

### Create/Edit Dialog

| State | Visual Treatment |
|-------|------------------|
| Open (create) | All switches OFF, empty name/description |
| Open (edit) | Switches pre-populated from role data, name/description filled |
| Submitting | Primary CTA shows `Loader2` spinner + disabled state, all inputs disabled |
| Validation error | Red text below input field: `text-destructive text-sm` |

### Delete Confirmation (AlertDialog)

| State | Visual Treatment |
|-------|------------------|
| Open | AlertDialog with role name in body text |
| Confirming | "Eliminar rol" button shows Loader2 spinner + disabled |
| Error | toast.error with server message |
| Success | Dialog closes, table refreshes, toast.success |

### Permission Toggles

| State | Visual Treatment |
|-------|------------------|
| ON | Switch in primary color (Shadcn default) |
| OFF | Switch in muted state (Shadcn default) |
| Disabled | Never — all switches are always interactive in the dialog |

## Responsive Behavior

This phase follows the project's existing responsive patterns:

| Breakpoint | Behavior |
|------------|----------|
| Desktop (1024px+) | Full table with all 5 columns visible, sidebar expanded |
| Tablet (768-1023px) | Full table, sidebar collapsed to icon mode |
| Mobile (<768px) | Table horizontal scroll, sidebar as sheet overlay |

No special responsive handling needed beyond what Shadcn Table + Sidebar already provide. The dialog uses `DialogContent` which is responsive by default (max-width with padding).
