### Gaps Summary

One gap is blocking complete confidence in goal achievement: the backend has a TypeScript compilation error in production code (`users.controller.ts` TS1272). The fix is a one-line import change: `import { CurrentUser, JwtUser }` → `import { CurrentUser, type JwtUser }`. The spec file errors (3 calls with wrong argument count) are test-only and excluded from the production build.

The error is isolated, mechanical to fix, and does not affect the runtime behavior of the application (NestJS transpiles separately). However, it violates the plan's acceptance criterion that "backend compiles cleanly with zero TypeScript errors" and could block Railway deployment if the CI pipeline uses the full `tsconfig.json` (not `tsconfig.build.json`).
