# Phase 12.5 Plan 02: Backend Service Specs (Products, Supplies, TiendanubeConfig) Summary

**One-liner:** Behavior-focused unit specs for three domain services with mocked repositories — SKU/BOM logic, supply status guards, and rate string→number parsing.

## What Was Built

Three `.spec.ts` files covering the most complex untested backend services, each following the established `Test.createTestingModule` pattern from the existing spec files.

### products.service.spec.ts (11 tests)

- `describe('create')`: SKU concatenation assertion (`BIL.HEF.LIS.MAR.U` = type + name + finish + color + size skuCode join) + 23505→ConflictException + NotFoundException when catalog dimension missing
- `describe('updateBom')`: deactivates old entries via `manager.update` + inserts new active set + ConflictException when supply inactive + NotFoundException when supply missing + empty BOM deactivates all
- `describe('addPrice')`: append-only — creates new record via priceHistoryRepo.create/save + NotFoundException when product missing

### supplies.service.spec.ts (10 tests)

- `describe('toggleStatus')`: ConflictException when deactivating supply in active BOM (bomRepo.count > 0) + ConflictException when reactivating supply with inactive supplier (before BOM check) + happy deactivate (count = 0)
- `describe('create')`: transaction path when initialPrice provided (entityManager.transaction called, mockManager.create/save used) + no-transaction path when initialPrice absent (supplyRepo.create/save called directly, currentPrice = null)
- Supporting: NotFoundException for missing type and missing supplier

### tiendanube-config.service.spec.ts (10 tests)

- `describe('updateGatewayRate')`: NotFoundException when gateway absent (gatewayRateRepo.save not called) + parses ratePercent string→number (TypeORM DECIMAL returns string, parseGatewayRate converts it)
- `describe('getTaxConfig')`: returns null when taxConfigRepo.findOne returns null + returns parsed numeric ivaRate/iibbRate when config exists
- `describe('updateTaxConfig')`: append-only — taxConfigRepo.create called with string values, parsed result has numeric ivaRate/iibbRate
