## What Was Built

- **Credentials provider** in `nemea-front/src/auth.ts`: added `import Credentials from 'next-auth/providers/credentials'`; registered alongside Google (additive, Google untouched). `authorize()` fetches `POST /api/auth/demo-login` using the Plan 01 backend contract, returns `{ backendToken, permissions, userId, email }` on success, `null` on any failure (never throws — T-12.5-AUTH).
- **signIn callback** updated: `provider === 'credentials'` returns true.
- **jwt callback** updated: added `user` param; credentials branch reads `user.backendToken/permissions/userId` into token. Google branch unchanged. No provider field added to token (T-12.5-03).
- **session callback**: unchanged — already reads from `token.backendToken`, `token.permissions`, `token.userId`.
- **DemoLoginButton.tsx**: Server Component with inner `'use server'` Server Action `handleDemoLogin` calling `signIn('credentials', { email: 'demo@nemea.app', redirectTo: '/' })` from `@/auth`. No `next-auth/react` import (no SessionProvider in (auth) layout). Label exactly "Entrar como demo" (D-02).
- **login/page.tsx**: imports `DemoLoginButton`; renders it conditionally behind `process.env.NEXT_PUBLIC_DEMO_LOGIN_ENABLED === 'true'` with a divider "o" above. Google form untouched; demo button placed secondary (D-02).
