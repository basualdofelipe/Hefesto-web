**2. [Rule 2 - Adaptation] CalculadoraClient test uses getAllByPlaceholderText + index**
- **Found during:** Task 1 forward mode test

- **Fix:** Use `screen.getAllByPlaceholderText('0')[0]` to target the first input (price/gain)

**3. [Rule 2 - Adaptation] ScenarioEditorClient mock uses bare function for next/link**
- **Found during:** Task 1 first ScenarioEditorClient render
- **Issue:** `jest.mock('next/link', () => ({ default: fn }))` caused "Element type is invalid: got object" — the component's `import Link from 'next/link'` resolves differently in Jest CJS transform
- **Fix:** Use `jest.mock('next/link', () => { const Link = fn; return Link; })` (module is the function itself)

**4. [Rule 2 - Adaptation] Switch component mocked for jsdom compatibility**
- **Found during:** Task 1 ScenarioEditorClient render
- **Issue:** Radix `Switch` from `@/components/ui/switch` caused element type error under jsdom
