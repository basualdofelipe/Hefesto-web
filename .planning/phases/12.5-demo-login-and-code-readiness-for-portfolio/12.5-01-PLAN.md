## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12.5-01 | Elevation of Privilege | demo-login endpoint reachable in prod | mitigate | `DEMO_LOGIN_ENABLED` Joi default `false`; `validateDemoLogin` rejects (UnauthorizedException) before any DB lookup when flag !== 'true' — backend authoritative even if a frontend calls it |
| T-12.5-ID | Information Disclosure | demo email enumeration | mitigate | Both "flag off" and "user not found" return 401 UnauthorizedException; no differential status code or response shape distinguishing user presence |
| T-12.5-MA | Tampering | mass assignment on DemoLoginDto | mitigate | DTO declares only `email` (`@IsEmail()`); global `ValidationPipe` whitelist strips extra fields |
| T-12.5-03 | Spoofing | JWT session confusion across providers | mitigate | JWT payload is exactly `{ sub, email, permissions }` — identical to Google path, no provider field; `PermissionsGuard`/`@RequirePermission` untouched |
| T-12.5-SC | Tampering | npm/pip/cargo installs | accept | Zero new dependencies this phase (SPEC constraint); no install tasks — supply-chain surface unchanged |
</threat_model>
