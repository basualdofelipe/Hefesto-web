## What Was Built

- **DemoLoginDto** (`nemea-back/src/auth/dto/demo-login.dto.ts`): Single `email!: string` field with `@IsEmail()` and `@ApiProperty`. No other fields — mass-assignment defense (T-12.5-MA).
- **DEMO_LOGIN_ENABLED** in `env.validation.ts`: `Joi.string().valid('true', 'false').default('false')` — optional with default, never `.required()` (prevents boot failure on existing deployments, T-12.5-01).
- **validateDemoLogin()** in `auth.service.ts`: reads flag via `configService.get()` (not `getOrThrow`), env-gate BEFORE DB lookup, mints `{ sub, email, permissions }` JWT identical to Google path (T-12.5-03), returns `AuthResponseDto` with `name/pictureUrl` null-coalesced.
- **POST demo-login** in `auth.controller.ts`: `@Public() @HttpCode(200)`, decorator ordering matches `googleLogin`, delegates to `validateDemoLogin`.
- **4 new test cases** in `auth.service.spec.ts`: flag=true+user exists (success), flag=false (DB not called), flag=unset, user-not-found (jwtService.sign not called).
