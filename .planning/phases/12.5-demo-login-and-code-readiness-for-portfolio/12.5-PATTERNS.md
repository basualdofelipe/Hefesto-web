### `nemea-back/src/auth/auth.service.ts` ‚Äî add `validateDemoLogin()` (service, request-response)

**Analog:** `validateGoogleToken` in `nemea-back/src/auth/auth.service.ts` lines 25‚Äì62

**Imports already present** (lines 1‚Äì8) ‚Äî no new imports needed. `configService` is already injected at line 18.

**Core pattern ‚Äî new method to append after `getProfile()`:**
```typescript
async validateDemoLogin(email: string): Promise<AuthResponseDto> {
  // Step 1: env-gate ‚Äî use configService.get (NOT getOrThrow ‚Äî field is optional)
  const enabled = this.configService.get<string>('DEMO_LOGIN_ENABLED');
  if (enabled !== 'true') {
    throw new UnauthorizedException('Demo login no disponible');
  }

  // Steps 2‚Äì4: identical to validateGoogleToken lines 32‚Äì61, minus Google steps
  const user = await this.usersService.findActiveByEmail(email);
  if (!user) {
    throw new UnauthorizedException('Usuario demo no encontrado');
  }

  const permissions = extractPermissions(user.role);  // user.role is eager-loaded
  const accessToken = this.jwtService.sign({
    sub: user.id,
    email: user.email,
    permissions,
  });

  return {
    accessToken,
    user: {
      id: user.id,
      email: user.email,
      permissions,
      name: user.name ?? null,
      pictureUrl: user.pictureUrl ?? null,
    },
  };
}
```

**Critical:** `configService.get()` (not `.getOrThrow()`) because `DEMO_LOGIN_ENABLED` is optional. The existing `GOOGLE_CLIENT_ID` uses `.getOrThrow()` at line 21 ‚Äî do NOT copy that pattern here.
