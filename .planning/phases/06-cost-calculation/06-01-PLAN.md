  <behavior>
    - calculateAll() returns Map<string, ProductCostData> with correct cost for products with BOM
    - Product without BOM entries returns no entry in the map (cost is null on the controller side)
    - Product with supply that has no price history: cost is partial, warning includes "{supplyName} sin precio"
    - Product with inactive supply in BOM: cost includes the supply's last price, warning includes "{supplyName} inactivo"
    - Decimal precision: cost rounds to 2 decimal places via Math.round(value * 100) / 100
    - calculateForProduct(id) returns a single ProductCostData or null
  </behavior>
  <action>
    1. Create `nemea-back/src/costs/dto/product-with-cost.dto.ts` with:
       - `CostBreakdownItem` interface: supplyId, supplyName, supplyType, quantity, unitType, unitPrice (number|null), lineCost (number|null), isSupplyActive
       - `ProductCostData` interface: cost (number), costBreakdown (CostBreakdownItem[]), costWarnings (string[])
       - `ProductWithCost` interface extending ProductWithPrice: adds cost (number|null), costBreakdown (CostBreakdownItem[]|null), costWarnings (string[])

    2. Create `nemea-back/src/costs/costs.service.ts`:
       - Inject `@InjectRepository(SuppliesPerProductHistory)` as bomRepo and `@InjectRepository(SupplyPriceHistory)` as priceHistoryRepo
       - `calculateAll()`: 2 queries + in-memory calc as shown in RESEARCH.md code example
         - Query 1: `bomRepo.find({ where: { isActive: true }, relations: ['product', 'supply'] })`
         - Query 2: `priceHistoryRepo.query('SELECT DISTINCT ON (supply_id) supply_id, price FROM supply_price_history ORDER BY supply_id, created_at DESC')`
         - Build priceMap (supplyId -> price as number via parseFloat)
         - Group BOM entries by product.id
         - For each product: compute breakdown items with lineCost = Math.round(unitPrice * quantity * 100) / 100
         - Track warnings for supplies without price and inactive supplies
         - Return Map<string, ProductCostData>
       - `calculateForProduct(productId: string)`: same 2-query approach but filtered to single product
         - Query 1: bomRepo.find with additional `product: { id: productId }` filter
         - Query 2: only fetch prices for supply IDs found in BOM (use `WHERE supply_id IN (...)`)
         - Return ProductCostData | null

    3. Create `nemea-back/src/costs/costs.module.ts`:
       - Import `TypeOrmModule.forFeature([SuppliesPerProductHistory, SupplyPriceHistory])`
       - Provider: CostsService
       - Export: CostsService
       - Do NOT import ProductsModule or SuppliesModule (avoids circular deps per Pitfall 2)

    4. Register `CostsModule` in `nemea-back/src/app.module.ts` imports array (before ProductsModule)

    5. Write unit tests in `costs.service.spec.ts`:
       - Mock bomRepo.find and priceHistoryRepo.query
       - Test: product with 2 BOM items returns correct total cost
       - Test: product without BOM returns no entry in map
       - Test: supply without price returns partial cost + warning
       - Test: inactive supply included in cost + warning
       - Test: decimal rounding (e.g., 1234.567 rounds to 1234.57)
